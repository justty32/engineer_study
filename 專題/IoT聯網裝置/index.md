## 專題目標與情境

做一塊 **IoT 聯網裝置電路板**，並為它寫韌體。情境設定：

- **板上有實體電源**（外部供電 / 電池 → 板上電源樹）。
- **主控是小型 ARM MCU**（Cortex-M 級），韌體用 **C** 撰寫。
- **聯網交給外購的通訊模組**：模組本身處理 TCP/IP、Wi-Fi、NB-IoT 等協定堆疊；MCU 透過介面（多半是 UART + AT 指令，或 SPI/SDIO）驅動它收送資料。
- **擁有者實際負責的兩件事：① 設計電路板（硬體）、② 寫韌體（軟體）。**

因此本專題的重心放在**板級硬體設計**與**韌體—模組整合**這兩個實作面，而非通訊理論本身（理論在通訊特化已具備）。

## 系統分層（這塊板子長什麼樣）

```
[ 感測器 / 致動器 ] ──(GPIO/ADC/I²C/SPI)──┐
                                          │
[ 電源：外部/電池 ]──[電源樹: 保護→DC-DC/LDO→各域]──┐
                                          │        │
                                  [ ARM MCU (韌體, C) ]
                                          │
                       (UART+AT / SPI / SDIO 介面)
                                          │
                              [ 外購通訊模組 ]
                          (Wi-Fi / NB-IoT, 內建 TCP/IP/TLS)
                                          │
                                ((天線 / RF 走線))
                                          │
                                      [ 基地台 / AP ]──→[ 雲端 (MQTT/HTTP) ]
```

擁有者的工作邊界：上圖中**電源樹、MCU 周邊接線、模組介面、天線/RF 走線**屬電路板設計；**MCU 內跑的所有程式 + 與模組的通訊協定**屬韌體。
