## 第 1 章　架構抉擇：裸機 vs RTOS

- **裸機 superloop + 中斷**：`while(1)` 主迴圈輪詢 + 中斷處理即時事件。簡單、省資源、好預測。
  - 關鍵：主迴圈**不可 blocking**（不要 `delay()` 空轉）；用**協作式狀態機 / event loop**，每次迴圈推進一點。
  - 適合：單純週期任務、資源吃緊的小 MCU。
- **RTOS（如 FreeRTOS）**：多任務、阻塞式 API 好寫（`xQueueReceive` 等）、優先權排程。
  - 代價：每任務一份堆疊（RAM）、要處理同步（mutex/semaphore）與優先權反轉。
  - 值得用的時機：多個獨立時序任務（取樣 / 連線 / UI）、阻塞式流程多、團隊協作。
- 抉擇細節與排程/同步原理 → [E2 即時作業系統](../../電機/09-嵌入式特化/E2-即時作業系統.md)。對本專題：連線（AT 引擎）與感測常用 RTOS 分任務較清爽；極簡省電裝置用 superloop 也夠。

## 第 2 章　分層與狀態機

分層（與軟體分層直覺一致，從簡）：

```
應用邏輯 ── OTA / 設定 / log
   └ 中介：AT 引擎、感測器服務、儲存
       └ HAL / 驅動：UART、SPI、I²C、ADC、Timer、WDT、RTC
```

- **裝置行為用狀態機描述**：`開機 → 自檢 → 連線 → (週期: 取樣→打包→上傳) → 睡眠 → 喚醒 …`。
- **事件驅動**：事件來自中斷、計時器、模組 URC；主迴圈/任務消費事件佇列。
- 模組化、邊界清楚 → 好測試、好移植。

## 第 3 章　非阻塞原則

- 主流程**絕不 busy-wait**（等 ADC、等 `OK`、等延遲）。改用：計時器到期旗標、中斷回呼、DMA 完成通知、RTOS 佇列阻塞。
- 長動作切成狀態機步進，或丟給中斷/DMA 在背景做，CPU 趁機睡。

## 第 4 章　低功耗設計（電池壽命的決定因素）

IoT 電池壽命主要由**睡眠時的平均電流**決定（量化見 [電源與電池 §8](硬體-電源與電池.md#第-8-章功耗與電量預算)）。核心策略：**盡量待在最深的睡眠，事件來才醒，做完馬上回睡（duty cycling）。**

### 4.1 MCU 功耗模式
- **Run**：全速跑。
- **Sleep**：CPU 停、周邊與時脈仍在，喚醒最快。
- **Stop**：時脈大多關閉、保留 RAM、可由少數源喚醒，省很多。
- **Standby / Shutdown**：幾乎全關、RAM 多半不保留、喚醒像重開機，最省但代價高。
- 取捨：睡得越深越省，但**喚醒時間與要保留的狀態**越受限。依喚醒頻率挑模式。

### 4.2 喚醒源
RTC 鬧鐘（週期任務）、GPIO 中斷（按鍵 / 感測器 / 模組 `RI`）、LPTIM、低功耗比較器。設計時想清楚「誰把我叫醒」。

### 4.3 把不用的關掉
- **關電源域**：用 GPIO 控制 LDO 的 EN，睡眠時切斷感測器 / 模組（見 [電源與電池](硬體-電源與電池.md)）。
- 關不用的周邊時脈、把閒置 GPIO 設定為不漏電狀態（避免浮接 / 上拉漏電）。
- 用 **LPUART**、低功耗 RTC 等專為睡眠設計的周邊。

### 4.4 讓 CPU 多睡
- **DMA 搬資料**（UART/ADC）讓 CPU 睡，搬完才喚醒。
- 降頻 / 動態調頻；非必要時用內部 RC 振盪器（省晶振起振）。

### 4.5 量測導向最佳化
**先量電流剖面，再優化**。常見耗電大戶：通訊發射、未關的感測器、LED、上拉電阻漏電、LDO 靜態電流。工具與方法見 [電源與電池 §9](硬體-電源與電池.md#第-9-章量測)。

## 第 5 章　可靠度

- **看門狗（WDT）**：主迴圈 / 監督任務定時餵狗，卡死自動重啟。
- **電壓偵測（BOR / brown-out reset）**：低電壓時安全重置，避免半磚行為。
- 開機自檢、進入已知安全狀態、錯誤計數與回報。

## 第 6 章　設定與資料持久化

- 用內部 flash / 外接 EEPROM / flash 存設定與緩存資料。
- **斷電安全**：寫入採雙區 + 校驗 / journaling，避免寫一半斷電壞檔；注意 flash **磨損平衡**與寫入次數限制。

## 第 7 章　OTA 與版本

- **A/B 雙分區 + bootloader**：新韌體寫到備援區，驗證（簽章 / CRC）通過才切換，失敗**回滾**。
