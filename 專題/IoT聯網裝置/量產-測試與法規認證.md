# 量產：測試與法規認證

> 專題《IoT 聯網裝置》專題特有筆記（待產出清單第 5 份）。專題總覽見 [index](index.md)。
> 名詞對照見 [專題中英名詞對照表 → 量產測試與認證](中英名詞對照表.md#量產測試與認證)。

> 這塊常被工程師忽略到最後才爆炸：**認證又貴又久、量產測試要在設計階段就鋪路**。越早規劃越省。

## 第 1 章　從打樣到量產的階段

- **EVT（工程驗證）**：第一版板，驗功能、抓設計錯誤。
- **DVT（設計驗證）**：修正後驗性能、可靠度、跑認證前測。
- **PVT（量產驗證）**：用量產治具/流程小批試產，驗良率與產測。
- **MP（量產）**：正式生產。

每階段都該有明確的進入/退出條件與問題清單。

## 第 2 章　設計可測試性（DFT）

在**設計階段**就要為產線著想：
- 留**測試點**（電源軌、關鍵訊號）、**SWD/JTAG** 燒錄與除錯腳、飛針可達的焊盤。
- 規劃**產測韌體（test firmware）**：上電自檢、逐一點亮/量測周邊、回報結果。
- 序號 / MAC / 憑證的**寫入（provisioning）** 流程要設計好。

## 第 3 章　產線測試

- **ICT（在線測試）/ 飛針**：量元件值、抓短路 / 斷路 / 缺件 / 反向。
- **FCT（功能測試）**：上電跑產測韌體，量電壓電流、測各周邊與接口是否正常。
- **燒錄 + provisioning**：燒 bootloader / 應用韌體，寫入唯一序號、MAC、金鑰 / 憑證。
- **RF 測試 / 校正**：量發射功率、頻率、頻譜、接收靈敏度；常在**屏蔽箱**內用綜測儀進行。
- **老化（burn-in）**：高溫 / 通電一段時間抽測，篩早夭品。
- **可追溯**：序號 + 條碼 + 產測 log，便於 RMA 與良率分析。

## 第 4 章　法規認證（無線裝置必過）

無線產品上市前須通過銷售地的法規認證：

- **各地主管 / 標誌**：美 **FCC**、歐 **CE（RED 指令）**、台 **NCC**、日 MIC、中 SRRC、韓 KC…
- **測試項目**：
  - **EMC**：輻射 / 傳導發射（你別吵到別人）、抗擾度（別人別吵到你）。
  - **RF / 無線電**：發射功率、佔用頻寬、頻譜遮罩、雜散發射。
  - **SAR**：貼身 / 近人體裝置的電磁波比吸收率。
  - **安規（safety）**：電氣安全（尤其含市電 / 電池）。
- **環保 / 材料**：RoHS（有害物質）、REACH、無鉛製程。

## 第 5 章　模組「預認證」的價值（為何買現成模組）

這是 IoT 普遍外購通訊模組的關鍵理由：
- 買**已通過認證的模組（pre-certified / modular approval）**，其無線部分的認證可被整機沿用或大幅簡化，你**主要只需做整機 EMC** 與安規，省下昂貴的完整射頻認證。
- 代價/限制：
  - 認證通常**綁定特定天線**（廠商給可用天線增益清單）；**換天線可能要重測**。
  - 須遵守模組廠的整合條件（佈局、屏蔽、標示）。
- 結論：**選模組時就先確認它的認證涵蓋你的目標市場與頻段**，能省最多時間與金錢。

## 第 6 章　電信入網（蜂巢式 / NB-IoT 特有）

蜂巢式裝置除了上面的法規，還要過電信相關認證：
- **PTCRB / GCF**：模組 / 裝置的電信產業認證。
- **電信商認證（carrier certification）**：部分電信商要求裝置通過其自家測試才准上網。
- **SIM / eSIM、APN、頻段（band）**：確認模組支援當地電信使用的頻段與核心網設定。
- NB-IoT 還涉及覆蓋、PSM/eDRX 行為與電信商配置的相容性。

## 第 7 章　文件與標示

- **合規聲明（DoC）**、技術文件、測試報告存檔。
- 產品標示：認證標誌（FCC ID / CE / NCC 編號）、序號、警告。
- 使用手冊、安全資訊。

## 第 8 章　時程與成本提醒

- 認證**動輒數週～數月、費用可觀**，且常需改板重測——**在 schematic / 選料階段就把認證策略定下來**（選預認證模組、預留 EMC 對策、留天線匹配與屏蔽位置）。
- 把認證排進專案時程的關鍵路徑，別等量產前才送測。
