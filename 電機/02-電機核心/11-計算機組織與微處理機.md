> 名詞對照見 [中英名詞對照表 → 計算機組織與微處理機](../中英名詞對照表-2.md#計算機組織與微處理機)。

## 這科在學什麼

寫程式跑在 CPU 上，這科把 CPU 拆開來看：怎麼用邏輯閘 + 暫存器組成一台「會跑指令」的機器。內容包含 ISA、資料路徑、控制路徑、管線、記憶體階層、I/O、效能評估。
身為軟體強項者，重點放在「跟軟體的對應」與「為什麼硬體要這樣做」。

## 第 1 章　計算機抽象層

從上而下：
1. 應用程式（C++、Python）
2. OS、編譯器
3. 組合語言 / 機器碼（ISA）
4. 微架構（管線、執行單元）
5. RTL（暫存器轉移層）
6. 邏輯閘
7. 電晶體
8. 物理（半導體）

**ISA**（Instruction Set Architecture）是軟硬體之間的契約——介於 3 與 4 之間。

## 第 2 章　數值表示

- 整數：補數表示（無號 / 有號）。
- 浮點數：**IEEE 754**，sign + exponent + mantissa；單精度 32 bit、雙精度 64 bit。注意非正規數、NaN、Inf 等特例。
- 字元：ASCII / Unicode。
- 大小端（endianness）：little vs big。

## 第 3 章　指令集架構（ISA）

### 3.1 分類
- **RISC**（精簡指令集）：少而簡單、固定長度。例：MIPS、RISC-V、ARM（Aarch64 偏 RISC）。
- **CISC**（複雜指令集）：指令多、長度不固定。例：x86 / x86-64。現代 x86 內部解碼為 micro-ops，跑在類 RISC 微架構上。

### 3.2 主要指令類型
- **資料搬移**：load / store / move
- **算術 / 邏輯**：add、sub、and、or、shift
- **控制流**：branch、jump、call、return
- **特權 / 系統指令**：interrupt、syscall

### 3.3 定址模式
立即、暫存器、絕對、相對、間接、基址 + 偏移、索引等。

## 第 4 章　資料路徑與控制

### 4.1 單週期 CPU
一個時脈完成一條指令：取指 → 解碼 → 執行 → 存回。
資源使用率低、時脈受最慢指令拖累。

### 4.2 五階段管線（pipeline）
最經典的 MIPS 結構：
1. **IF**（取指）
2. **ID**（解碼 + 讀暫存器）
3. **EX**（執行 / ALU）
4. **MEM**（記憶體存取）
5. **WB**（寫回）

每個時脈周期都能「啟動」一條指令，吞吐量大增。

### 4.3 管線冒險（Hazards）
- **結構冒險**：資源衝突。
- **資料冒險**：相依（RAW）。解法：旁路（forwarding）、stall（插入泡）。
- **控制冒險**：分支。解法：分支預測（branch predictor）、延遲槽（早期 RISC）。

### 4.4 現代高階特性
- **超純量（superscalar）**：多條管線並行。
- **亂序執行（out-of-order）**：依資料相依重排，配合 register renaming。
- **推測執行（speculative）**：先猜後驗，效能高但有安全議題（Spectre/Meltdown）。
- **SIMD**（SSE、AVX、NEON）：向量化指令一次處理多個資料。

## 第 5 章　記憶體階層

### 5.1 階層
暫存器 < L1 cache < L2 < L3 < 主記憶體 < SSD/HDD < 網路儲存。
越上層越快越貴越小。

### 5.2 快取（Cache）
利用**時間 / 空間區域性**：
- **直接對映、組相聯、全相聯**。
- **替換策略**：LRU、LFU、Random。
- **寫策略**：write-through vs write-back；write-allocate vs no-write-allocate。
- **快取失誤類型**：強制失誤（compulsory）、容量失誤（capacity）、衝突失誤（conflict）。

### 5.3 虛擬記憶體
- **頁式（paging）**：把實體分頁，行程看到自己的虛擬位址空間。
- **TLB**（Translation Lookaside Buffer）：快取 page table 翻譯。
- **頁缺失（page fault）**：訪問未在 RAM 的頁，OS 從磁碟載入。

### 5.4 一致性（Cache Coherence）
多核心系統下要保證所有核心看到一致的記憶體。常用 **MESI 協定**（Modified / Exclusive / Shared / Invalid）。

## 第 6 章　I/O 與匯流排

### 6.1 I/O 方式
- **輪詢（polling）**：CPU 不斷查狀態。簡單但浪費。
- **中斷（interrupt）**：硬體事件觸發 CPU 跳轉處理。
- **DMA（直接記憶體存取）**：另一個控制器代搬資料，CPU 只下指令。

### 6.2 匯流排
- 系統匯流排（CPU ↔ 記憶體）
- 周邊匯流排：PCI / PCIe、USB、SPI、I²C、UART、CAN。
  - 嵌入式系統用 SPI/I²C/UART 與感測器、IC 溝通。
  - PCIe 是現代高速擴充（GPU、SSD、網卡）。

## 第 7 章　效能評估

### 7.1 衡量
$$\text{執行時間}=\dfrac{\text{指令數}}{\text{程式}}\times\dfrac{\text{時脈周期}}{\text{指令}}\times\dfrac{\text{時間}}{\text{時脈周期}}=\text{IC}\times\text{CPI}\times T_c$$
（CPI = Cycles Per Instruction）

### 7.2 Amdahl 定律
若程式中某段佔比例 $p$ 被加速 $s$ 倍：
$$\text{總加速} = \dfrac{1}{(1-p)+\dfrac{p}{s}}$$
意義：不能被加速的部分最終會主導。

### 7.3 平行架構
- **指令級平行（ILP）**：超純量、亂序。
- **資料級平行（DLP）**：SIMD、GPU。
- **執行緒級平行（TLP）**：multi-core、SMT（hyperthreading）。
- **任務級平行**：分散式系統。

## 第 8 章　典型微處理器與微控制器

- **通用 CPU**：x86、ARM、RISC-V。
- **MCU（微控制器）**：STM32（Cortex-M）、AVR、ESP32、PIC ——含 CPU + 記憶體 + I/O，用於嵌入式系統。
- **DSP**：TI C6000、ADI SHARC，最佳化 MAC 等運算。
- **GPU**：高度平行向量處理器；現代 AI 加速主流。
- **NPU / TPU**：專用於神經網路推論的加速器。
- **FPGA**：可重組邏輯，介於 CPU 與 ASIC 之間。

## 第 9 章　組合語言與 CPU 的對應（極簡）

範例（RISC-V）：
```
add  x3, x1, x2     # x3 = x1 + x2
lw   x4, 0(x5)      # x4 = *(x5 + 0)
sw   x4, 4(x6)      # *(x6 + 4) = x4
beq  x1, x2, label  # if x1 == x2 jump to label
```
所有高階程式經編譯器後，都會落到這種層級的指令序列。
