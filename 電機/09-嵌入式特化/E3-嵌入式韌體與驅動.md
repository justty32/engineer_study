## 這科在學什麼

E1 建立硬體心智、E2 加上即時排程；本科談**韌體工程實務**：如何把硬體包裝成可重用的驅動與軟體層、通訊協定怎麼實作、開機與更新流程、以及讓產品可量產 / 可維護的工程習慣。

1. 韌體分層架構（暫存器 → HAL → 驅動 → 應用）
2. 周邊驅動與通訊協定（UART/SPI/I²C/CAN/USB）

## 第 1 章　韌體分層架構

```
應用 / 業務邏輯
   │
中介層（協定堆疊、檔案系統、RTOS）
   │
裝置驅動（device drivers）
   │
HAL（硬體抽象層）/ 暫存器存取（CMSIS）
   │
硬體（MCU 周邊暫存器）
```

- **HAL**：把「設定某暫存器某位元」包成 `HAL_UART_Transmit()` 之類的可攜 API。
- **驅動**：管理一個周邊 / 外部裝置的狀態與資料流。
- 分層的目的：可攜（換 MCU 只改 HAL）、可測、可維護。

## 第 2 章　驅動模型

### 2.1 驅動的三種 I/O 模式
- **輪詢（polling）**：簡單，CPU 忙等，適合短/ 低速。
- **中斷驅動（interrupt-driven）**：事件來才處理，效率高。
- **DMA**：大量 / 高速資料不佔 CPU。

實務常組合：例如 UART 收用中斷 + DMA，環形緩衝區（ring buffer）暫存。

### 2.2 阻塞 vs 非阻塞
- 阻塞 API：呼叫者等到完成（搭配 RTOS 讓出 CPU）。
- 非阻塞 / 回呼（callback）/ 事件：高吞吐、不卡任務。

### 2.3 環形緩衝區（Ring Buffer）
生產者（ISR）/ 消費者（任務）解耦的核心資料結構；無鎖單生產單消費可做到很高效。

## 第 3 章　通訊協定實作

### 3.1 UART
- 非同步：雙方約定鮑率、資料位、停止位、同位。
- 無時脈線 → 鮑率誤差需 < ~2~3%。
- 常配 DMA + idle line 中斷收不定長封包。
- 上層常跑：AT 命令、Modbus、自訂封包（含起始 / 長度 / CRC）。

### 3.2 SPI
- 主從、全雙工、四線（SCLK/MOSI/MISO/CS）。
- 模式 0~3（CPOL/CPHA 時脈極性與相位）——對錯就讀到亂碼。
- 高速（數十 MHz），接 Flash、顯示、感測器、ADC。

### 3.3 I²C
- 兩線（SCL/SDA）、開汲極 + 上拉、多裝置定址（7/10-bit）。
- 起始 / 停止 / ACK / 時脈延展（clock stretching）。
- 常見坑：匯流排卡死（need recover）、上拉電阻值、位址衝突。

### 3.4 CAN / CAN-FD
- 車用 / 工業，差動、抗噪、多主、訊息優先權仲裁。
- 上層：CANopen、J1939、UDS（診斷）。

### 3.5 USB
- 列舉（enumeration）、端點、裝置類別（CDC 虛擬串列、HID、MSC 隨身碟、DFU）。
- 複雜但有現成堆疊（TinyUSB、廠商 middleware）。

### 3.6 通訊可靠度
- 訊框、逸出（escaping）、CRC / checksum、序號、ACK / 重傳、逾時。
- 自訂協定要從一開始就設計這些，否則現場除錯惡夢。

## 第 4 章　感測器與致動器驅動

- 讀 datasheet：暫存器地圖、初始化序列、時序需求。
- 校正（offset / gain）、濾波（去雜訊）、單位換算。
- 致動器：PWM 驅動馬達 / LED、H 橋、伺服訊號。
- 典型流程：上電 → 設定暫存器 → 等就緒 → 週期讀 → 轉換工程單位。

## 第 5 章　開機鏈與 Bootloader

### 5.1 為什麼要 bootloader
- 出廠後還能更新韌體（不必拆機接 JTAG）。
- 多階段開機、選擇 application image、回復（recovery）。

### 5.2 典型流程
```
Reset → Bootloader（在 Flash 開頭）
  → 檢查是否進更新模式（按鍵 / 命令 / 無有效 App）
  → 驗證 App 映像（CRC / 簽章）
  → 跳轉到 App（設向量表偏移 VTOR、SP、PC）
```

### 5.3 記憶體分割
Bootloader 區 + App 區（+ 雙區 A/B）+ 設定區，由連結腳本與 bootloader 約定。

## 第 6 章　韌體更新（OTA / DFU）

### 6.1 更新管道
- **DFU**：USB / UART / SWD 本地更新。
- **OTA（空中更新）**：Wi-Fi / BLE / 行動網路遠端更新（IoT 必備，E5 再談連網）。

### 6.2 安全與可靠更新
- **雙區（A/B）/ 雙 bank**：更新到備區，驗證後切換，失敗回滾（fail-safe）。
- **完整性 + 真實性**：CRC + 數位簽章（防竄改、防偽韌體）。
- **斷電安全**：更新中斷電不可變磚（brick）。
- **版本 / 相容性管理**。

## 第 7 章　儲存與設定

- **設定值持久化**：EEPROM / Flash 模擬 EEPROM / 專用區，含磨損平衡（wear leveling）。
- **檔案系統**：FatFs（SD 卡 / USB）、LittleFS / SPIFFS（NOR Flash，掉電安全 + 磨損平衡）。
- **日誌（log）**：環形紀錄供現場除錯 / 黑盒子。

## 第 8 章　時間、排程與韌體結構

### 8.1 無 OS 的結構
- **超級迴圈 + 中斷**。
- **協作式狀態機 / 事件迴圈**（如 protothreads、QP 框架）。
- **時間片 / 簡易排程器**。

### 8.2 有 RTOS（接 E2）
- 每個周邊 / 子系統一個任務 + 佇列。
- 驅動以 ISR + 任務協作實作。

## 第 9 章　工程實務與品質

### 9.1 可移植與抽象
- HAL / BSP（板級支援包）分離硬體相依。
- 介面（function pointer / vtable）讓上層不綁特定晶片。

### 9.2 測試
- **單元測試**：在 PC 上跑（把 HAL mock 掉）。
- **硬體在環（HIL）**：真硬體 + 自動化測試。
- 靜態分析（MISRA-C 規範，車 / 安全領域）。

### 9.3 除錯技巧
- SWD 中斷點、RTT / SWO trace、邏輯分析儀對協定波形、fault handler 解析。
- 重現性問題：時序競爭、未初始化、堆疊溢位、`volatile` 漏標。

### 9.4 程式風格
- 固定寬度型別（`uint8_t`…）、避免動態配置、明確錯誤碼、防禦式編程。
- 中斷安全、可重入（reentrancy）。

## 第 10 章　資安基礎（接 E5）

- 安全開機（secure boot）：只執行簽章韌體。
- 除錯介面鎖定（量產後關 JTAG / 設讀保護 RDP）。
- 金鑰存於安全元件 / OTP，韌體加密。
- 通訊加密（TLS / DTLS）、防回滾。
