# E1 嵌入式系統基礎（Embedded Systems Fundamentals）概論

> 嵌入式特化第 1 科。
> 名詞對照見 [中英名詞對照表 → 嵌入式](../中英名詞對照表-4.md#嵌入式)。
>
> 註：擁有者軟體基礎強，故「純程式設計」部分從簡；重點放在**硬體 / 時序 / 暫存器 / 中斷**這層軟硬交界，這是純軟體開發較少接觸的地方。

## 這科在學什麼

嵌入式系統 = 為特定功能而生、運算資源受限、常需即時反應、直接操作硬體的電腦系統。本科建立微控制器（MCU）的硬體心智模型：

1. MCU 架構（核心、記憶體、匯流排）
2. 周邊（GPIO、計時器、通訊介面、ADC/DAC）
3. 中斷與例外
4. 時序、時脈、功耗
5. 暫存器層級程式設計與工具鏈

## 第 1 章　嵌入式 vs 一般電腦

| 面向 | 一般電腦 | 嵌入式 |
|---|---|---|
| 目的 | 通用 | 專用 |
| 資源 | 充裕（GB 記憶體） | 受限（KB~MB） |
| OS | 必備、重型 | 可無 OS（bare-metal）或 RTOS |
| 即時性 | 多半不要求 | 常需硬即時 |
| I/O | 抽象化 | 直接碰暫存器 / 接腳 |
| 功耗 | 次要 | 常是第一考量（電池） |

## 第 2 章　處理器核心

### 2.1 MCU vs MPU vs SoC
- **MCU（微控制器）**：CPU + Flash + RAM + 周邊整合在一顆，bare-metal / RTOS。例：STM32、AVR、PIC、ESP32。
- **MPU（微處理器）**：只有 CPU，外接 DRAM / 儲存，跑完整 OS（Linux）。
- **SoC**：把 CPU、GPU、DSP、NPU、周邊整合，手機 / 邊緣裝置。

### 2.2 指令集架構（ISA）
- **ARM Cortex-M**（嵌入式主流）：M0/M0+（低階）、M3/M4（含 DSP）、M7（高效能）、M33（TrustZone 安全）。
- **RISC-V**：開放 ISA，崛起中（ESP32-C、各家 MCU）。
- 8/16-bit：8051、AVR、MSP430，仍用於超低成本 / 低功耗。

### 2.3 馮諾依曼 vs 哈佛
- **馮諾依曼**：指令與資料共用匯流排。
- **哈佛 / 修改型哈佛**：指令與資料分匯流排（Cortex-M 採用），可同時取指 + 取資料。

### 2.4 管線、流程
Cortex-M 多為 3 級管線（取指 / 解碼 / 執行）；無快取 / 簡單分支預測 → 時序較可預測（利於即時性）。

## 第 3 章　記憶體與位址空間

### 3.1 記憶體種類
- **Flash**：存程式碼 / 常數，非揮發，寫入慢、有抹寫次數限制。
- **SRAM**：執行期資料 / 堆疊 / 堆積，揮發、快。
- **EEPROM / 模擬 EEPROM**：少量設定值。
- 外接：QSPI Flash、SDRAM、SD 卡。

### 3.2 記憶體映射 I/O
周邊暫存器映射到位址空間，讀寫某位址即操作硬體。這是嵌入式的核心觀念——
```c
#define GPIOA_ODR (*(volatile uint32_t*)0x48000014)
GPIOA_ODR |= (1 << 5);   // 點亮接 PA5 的 LED
```
`volatile` 告訴編譯器此位址值會被硬體改變、不可最佳化掉。

### 3.3 記憶體配置
- **.text**（程式碼，Flash）、**.rodata**（常數）、**.data**（已初始化變數，開機由 Flash 複製到 RAM）、**.bss**（清零變數）。
- **stack**（向下成長）、**heap**（嵌入式常避免動態配置以防碎片）。
- 連結腳本（linker script）定義各段擺放位置。

## 第 4 章　開機流程（Bare-metal）

1. 重置 → CPU 從向量表載入初始 SP 與 Reset_Handler。
2. **啟動碼（startup）**：複製 .data 到 RAM、清 .bss、設定時脈。
3. 呼叫 `main()`。
4. （無 OS）`main` 內 `while(1)` 主迴圈 + 中斷服務。

## 第 5 章　周邊（Peripherals）

### 5.1 GPIO
- 輸入 / 輸出 / 替代功能 / 類比模式。
- 上拉 / 下拉、推挽 / 開汲極、驅動強度、施密特觸發。

### 5.2 計時器（Timer / Counter）
- 計數、輸入捕捉（量測脈衝寬 / 頻率）、輸出比較、**PWM** 產生（馬達 / LED 調光 / 電源）。
- 看門狗計時器（watchdog）：程式當掉自動重置。
- 系統滴答（SysTick）：RTOS / 延時基準。

### 5.3 類比
- **ADC**：取樣外界類比訊號（解析度、取樣率、SAR / Σ-Δ）。
- **DAC**：輸出類比。
- **比較器、運放**（部分 MCU 內建）。

### 5.4 通訊介面（E3 細談協定）
- **UART**：非同步串列、簡單、除錯常用。
- **SPI**：高速、全雙工、主從、多用於感測器 / Flash / 顯示。
- **I²C**：兩線、多裝置定址、低速感測器。
- **CAN**：車用 / 工業、抗噪、多主。
- **USB、Ethernet、SDIO**：高速 / 連網。

### 5.5 DMA（直接記憶體存取）
周邊 ↔ 記憶體搬資料不經 CPU，省週期、提升吞吐（如 ADC 連續取樣存陣列、UART 大量收發）。

## 第 6 章　中斷與例外

### 6.1 為什麼用中斷
輪詢（polling）浪費 CPU 又延遲；中斷讓硬體事件主動打斷 CPU 即時處理。

### 6.2 NVIC（Cortex-M 巢狀向量中斷控制器）
- 向量表：每個中斷源對應一個 ISR 位址。
- **優先權**：高優先可搶占低優先（巢狀）。
- 尾鏈（tail-chaining）、晚到搶占等硬體最佳化降低延遲。

### 6.3 ISR 撰寫原則
- **短、快、不阻塞**：不在 ISR 裡做耗時 / 等待 / 大量運算。
- 與主程式共享資料用 `volatile` + 臨界區（關中斷 / 原子操作）保護。
- 旗標 / 佇列把工作丟回主迴圈或 RTOS 任務（下半部）。

### 6.4 中斷延遲（Latency）
從事件發生到 ISR 開始執行的時間——即時系統的關鍵指標。受優先權、關中斷區段、管線影響。

## 第 7 章　時脈與時序

- **時脈樹**：晶振 / RC → PLL 倍頻 → 各匯流排 / 周邊分頻。
- 時脈設定錯誤是新手常見 bug（周邊不動、UART 鮑率錯）。
- **時序限制**：建立 / 保持時間、訊號完整性（接回 A3）。

## 第 8 章　低功耗設計

- **電源模式**：Run / Sleep / Stop / Standby，逐級關閉時脈與電源域。
- **喚醒源**：中斷、RTC、外部接腳。
- 策略：盡量睡、事件驅動、降頻、關閉未用周邊、DMA 代替 CPU。
- 量測：平均電流 × 時間決定電池壽命。

## 第 9 章　開發工具鏈與除錯

### 9.1 工具鏈
- 交叉編譯（cross-compile）：在 PC 上用 `arm-none-eabi-gcc` 產生目標碼。
- 連結腳本、`objcopy` 產生 .bin/.hex、燒錄。
- HAL / 暫存器庫（CMSIS）。

### 9.2 除錯
- **JTAG / SWD**：硬體除錯介面，設中斷點、單步、看暫存器 / 記憶體。
- **printf via UART / SWO / RTT**：輸出除錯訊息。
- 邏輯分析儀、示波器看實體訊號。
- **硬體故障（HardFault）**：解析 fault 暫存器找原因（非法位址、未對齊、堆疊溢位）。

## 第 10 章　設計考量與品質

- **可靠度**：看門狗、brown-out 偵測、ECC、冗餘。
- **記憶體安全**：MPU（記憶體保護單元）隔離、堆疊溢位偵測。
- **可測試性、可維護性**：模組化、HAL 抽象（E3）。
- **功能安全標準**：ISO 26262（車）、IEC 61508（工業）、DO-178C（航空）。

## 與其他科目的銜接

- 計算機組織（電機核心）：CPU / 記憶體 / 匯流排原理。
- 邏輯設計、電子學：周邊電路、訊號準位。
- 控制系統 / 馬達驅動：MCU 是控制器的實作平台（PWM、ADC、PID）。
- RTOS（E2）：在本科 bare-metal 之上加排程。
