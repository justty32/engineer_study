# E5 IoT 與邊緣運算（IoT & Edge Computing）概論

> 嵌入式特化第 5 科（嵌入式特化完結篇）。
> 名詞對照見 [中英名詞對照表 → 嵌入式](../中英名詞對照表-4.md#嵌入式)。

## 這科在學什麼

把單機嵌入式裝置接上網路、規模化、加上雲端與智慧，就是**物聯網（IoT）**與**邊緣運算（Edge Computing）**。本科整合前四科（硬體、RTOS、韌體、Linux）並接回通訊與計算機網路，談端到端的系統：

1. IoT 系統架構（端 / 邊 / 雲）
2. 連線技術（短距 / 低功耗廣域 / 行動）
3. IoT 通訊協定（MQTT、CoAP…）
4. 邊緣運算與邊緣 AI
5. 資安、佈建、維運

## 第 1 章　IoT 系統架構

```
感測 / 致動（端裝置）
   │  短距 / LPWAN
邊緣閘道 / 邊緣節點（聚合、預處理、邊緣 AI）
   │  IP 網路（Wi-Fi / 行動 / 乙太）
雲平台（資料儲存、分析、裝置管理、應用）
   │
應用 / 儀表板 / 商業邏輯
```

- **端（device / thing）**：MCU + 感測器 + 無線，低功耗。
- **邊（edge）**：閘道 / 邊緣伺服器，靠近資料源做運算（低延遲、省頻寬、隱私）。
- **雲（cloud）**：大規模儲存與分析、跨裝置協調。

## 第 2 章　為什麼要邊緣運算

純雲端架構的問題：
- **延遲**：來回雲端太慢（自駕、工控、AR）。
- **頻寬 / 成本**：影像 / 高頻感測全上傳不切實際。
- **可用性**：斷網仍要運作。
- **隱私 / 法規**：資料不出本地。

邊緣運算把運算下放到靠近資料的地方；雲端負責訓練模型、全域協調、長期儲存。**雲-邊-端協同**是主流。

## 第 3 章　連線技術

### 3.1 短距無線
- **BLE（低功耗藍牙）**：穿戴 / 信標 / 配對，Mesh 擴展。
- **Wi-Fi**：高頻寬，耗電較高；Wi-Fi HaLow（900 MHz）為 IoT 低功耗版。
- **Zigbee / Thread（802.15.4）**：低功耗 Mesh，智慧家庭（Matter 統一標準）。
- **UWB**：精準測距 / 定位。
- **NFC / RFID**：近場、無源標籤。

### 3.2 LPWAN（低功耗廣域）
長距 + 超低功耗 + 低速率，電池可數年：
- **LoRa / LoRaWAN**：免授權頻段、私有部署。
- **NB-IoT、LTE-M**：行動電信營運、授權頻段、覆蓋深。
- **Sigfox**：超窄頻。

### 3.3 行動與衛星
- 4G/5G：高頻寬、5G 的 mMTC（巨量機器通訊）與 URLLC（超可靠低延遲）（接無線通訊）。
- 衛星 IoT（LEO）：偏遠 / 海洋覆蓋。

### 3.4 選型權衡
頻寬 ↔ 功耗 ↔ 距離 ↔ 成本 ↔ 部署模式（自建 vs 電信）。

## 第 4 章　IoT 通訊協定

### 4.1 應用層
- **MQTT**：發布 / 訂閱、輕量、TCP 上跑，IoT 最常用；QoS 0/1/2、保留訊息、遺囑（LWT）。
- **CoAP**：類 REST、UDP 上跑、極輕量，受限裝置友善；可搭 DTLS。
- **HTTP / REST、WebSocket**：閘道 / 高階裝置。
- **AMQP**：企業級訊息佇列。

### 4.2 資料格式
- JSON（易讀、較大）、CBOR（二進位 JSON）、Protocol Buffers、MessagePack——受限裝置偏好緊湊二進位。

### 4.3 裝置管理協定
- LwM2M（OMA）、TR-069。
- 雲平台 SDK：AWS IoT Core、Azure IoT Hub、Google Cloud IoT。

### 4.4 互通標準
- **Matter**（over Thread / Wi-Fi）：智慧家庭跨品牌互通。

## 第 5 章　邊緣 AI（TinyML 與邊緣推論）

### 5.1 兩個層次
- **TinyML**：在 MCU（KB 級）上跑極小神經網路（關鍵字喚醒、手勢、異常偵測、震動診斷）。
- **邊緣 AI 加速**：在邊緣裝置（Jetson、NPU、邊緣 GPU）跑較大模型（影像辨識、物件偵測）。

### 5.2 模型最佳化
- **量化（quantization）**：浮點 → int8 / 更低，省記憶體 + 加速。
- **剪枝（pruning）、知識蒸餾**。
- 框架：TensorFlow Lite Micro、CMSIS-NN、microTVM、Edge Impulse、ONNX Runtime。

### 5.3 硬體加速
- MCU 內 DSP / NPU（Cortex-M55 + Ethos-U）。
- 邊緣 SoC NPU、邊緣 GPU、FPGA。

### 5.4 流程
資料收集 → 訓練（雲）→ 量化 / 轉換 → 部署到裝置 → 現場推論 →（可選）回傳改善。**聯邦學習**讓資料不出端也能協同訓練。

## 第 6 章　功耗與能源

- 端裝置常靠電池 / 能量採集（太陽能、振動、RF）。
- 策略：極致睡眠 + 事件喚醒、聚合上傳、LPWAN、duty cycling。
- 能量採集（energy harvesting）+ 超低功耗 MCU → 免電池 IoT。
- 電池壽命估算：平均電流 × 工作週期 vs 容量。

## 第 7 章　佈建、維運與裝置生命週期

- **佈建（provisioning）**：出廠注入身分 / 金鑰 / 憑證；首次連網安全註冊（zero-touch）。
- **OTA 更新**：韌體（E3）/ 系統映像（E4）/ 模型；A/B + 回滾 + 簽章。
- **監控（observability）**：遙測、健康狀態、遠端日誌、告警。
- **機群管理**：數萬裝置的批次更新、設定、停用。
- **退役**：金鑰撤銷、資料清除。

## 第 8 章　IoT 資安（重中之重）

IoT 是大型攻擊面（Mirai 殭屍網路即由弱密碼 IoT 組成）。

### 8.1 縱深防禦
- **裝置**：安全開機、安全元件 / TPM、除錯介面鎖定、防回滾、唯讀根。
- **通訊**：TLS / DTLS、雙向憑證認證、最小權限。
- **身分**：每裝置唯一身分 + 憑證；避免共用 / 預設密碼。
- **雲 / 後端**：存取控制、異常偵測、密鑰管理（KMS / HSM）。
- **更新**：簽章韌體、安全 OTA 管道。

### 8.2 法規與標準
- ETSI EN 303 645（消費 IoT 基線）、NIST、歐盟 CRA（網路韌性法案）。
- 禁用預設密碼、強制可更新性、漏洞揭露。

## 第 9 章　典型應用領域

- **智慧家庭**：燈控、門鎖、語音助理（Matter）。
- **工業 IoT（IIoT）/ 工業 4.0**：預測性維護、震動 / 溫度監測、數位分身。
- **智慧城市**：路燈、停車、環境感測、智慧電網（接電力特化）。
- **車聯網（V2X）**：車間 / 車路通訊。
- **醫療 IoT**：穿戴、遠距監測（接醫電特化）。
- **農業**：土壤 / 氣象、灌溉。
- **資產追蹤 / 物流**：LPWAN + GPS。

## 第 10 章　趨勢

- **AI 與邊緣融合**：生成式 / 大模型下沉到邊緣，端側推論。
- **數位分身（digital twin）**：實體裝置的雲端鏡像做模擬與最佳化。
- **5G/6G + IoT**：URLLC、mMTC、感測通訊一體（ISAC，接 A4/A5）。
- **可持續 IoT**：免電池、能量採集、低碳。
- **標準收斂**：Matter、統一資安法規。
