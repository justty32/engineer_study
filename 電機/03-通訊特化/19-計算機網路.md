# 計算機網路（Computer Networks）概論（精簡）

> 通訊特化第 6 科。風格：**精簡版**（軟體本科強項，重點在概念串接與通訊視角，不重述軟體工程細節）。
> 名詞對照見 [中英名詞對照表 → 計算機網路](../中英名詞對照表-2.md#計算機網路)。

## 一句話定位

通訊系統把「位元」可靠地送過一個鏈路；計算機網路把「鏈路」組成可路由、可定址、可端對端通訊的系統。本科是把通訊基礎抽到「協定堆疊」的層次。

## 第 1 章　分層模型

### 1.1 兩個模型對照
| OSI 7 層 | TCP/IP 4 層 | 典型協定 |
| --- | --- | --- |
| 應用 / 表現 / 會議 | 應用 | HTTP、DNS、TLS、SMTP |
| 傳輸 | 傳輸 | TCP、UDP、QUIC |
| 網路 | 網路 | IPv4 / IPv6、ICMP、BGP、OSPF |
| 資料連結 | 連結 | Ethernet、Wi-Fi (MAC)、PPP |
| 實體 | 連結（合在一起） | 銅線、光纖、無線（通訊層） |

OSI 教學用、TCP/IP 實務用。

### 1.2 分層的意義
封裝（encapsulation）：每層加自己的標頭。
解耦：每層只關心相鄰層介面，可獨立演化。

## 第 2 章　實體層（與通訊接合處）

通訊系統章節都講過：訊號、編碼、調變、頻寬、訊雜比。
這層在計網科是個「介面提供者」：
- 提供位元串給連結層。
- 規格：纜線、接頭、波形、時脈、線編碼。
- 範例：1000BASE-T 用 4D-PAM5、10GBASE-T 用 PAM16 + LDPC、光纖用 NRZ / PAM4 + FEC。

## 第 3 章　資料連結層

### 3.1 主要任務
- **介質存取控制（MAC）**：誰可以發。
- **訊框（framing）**：把位元切成 frame。
- **錯誤偵測 / 糾正**：CRC、FEC。
- **流量控制**：避免接收端被淹沒。
- **MAC 位址**：在區網內定址。

### 3.2 介質存取
- **無爭用**：時槽分配、令牌、輪詢。
- **隨機存取**：ALOHA、CSMA、CSMA/CD（古典乙太網）、CSMA/CA（Wi-Fi，先聽再發再退避）。

### 3.3 乙太網（Ethernet）
- 帧格式：前置碼 + Dest MAC + Src MAC + Type / Length + Payload + CRC。
- 演進：10 Mbps → 100 → 1G → 10G → 25/40/100/400G。
- 從共享匯流排（CSMA/CD）走到交換式（switched），現在幾乎都是 full-duplex。

### 3.4 交換器（Switch）
- 學習 MAC 表 → 把 frame 只送到目的埠。
- VLAN：邏輯分區。
- STP：避免迴圈造成廣播風暴。
- 現代資料中心交換器走 ASIC + SDN（OpenFlow）。

## 第 4 章　網路層

### 4.1 IP 定址
- **IPv4**：32 bit；CIDR 表示法 `192.168.0.0/16`。
- **IPv6**：128 bit；解決位址耗盡，附帶整潔的 SLAAC、ND、隱私延伸。
- 子網切割、NAT（地址轉譯）。

### 4.2 路由
- **路由協定**：
  - 域內（IGP）：OSPF（鏈路狀態，Dijkstra）、IS-IS、EIGRP。
  - 域間（EGP）：**BGP**（路徑向量，Internet 骨幹路由）。
- **路由表查詢**：最長前綴匹配 → 硬體上用 TCAM。

### 4.3 控制訊息
- ICMP：ping、traceroute、不可達回報。
- ARP（v4）/ NDP（v6）：把 IP 對到 MAC。

### 4.4 多播 / 廣播 / 任播
- 多播：IGMP / MLD + PIM。視訊串流節省頻寬。
- 任播：BGP + 多個位置同 IP，DNS 與 CDN 標配。

## 第 5 章　傳輸層

### 5.1 UDP
無連線、低延遲、無重傳。DNS、VoIP、即時遊戲、QUIC 底層。

### 5.2 TCP
連線導向、可靠傳輸（重傳 + 序號 + ACK）+ 流量控制（接收窗）+ 擁塞控制。
- **三次握手**：SYN / SYN-ACK / ACK。
- **擁塞控制**：
  - 經典：Tahoe / Reno（AIMD）。
  - 現代：CUBIC（Linux 預設）、BBR（Google，估測頻寬與 RTT）。
- 性能議題：高 BDP（bandwidth-delay product）下要增大窗、Selective ACK、Fast Retransmit。

### 5.3 QUIC（HTTP/3 底層）
跑在 UDP 上的新傳輸協定：
- 內建 TLS 1.3 加密。
- 一次握手即可開始送資料（0-RTT）。
- 多串流避免 HOL blocking。
- 連線遷移（IP 變動不斷線）。

### 5.4 SCTP、DCCP
多串流、訊息導向、行動 / 即時應用；普及度不如 TCP/UDP。

## 第 6 章　應用層（核心觀念，不細寫）

- **DNS**：階層式名稱服務；UDP 為主、加密用 DoT / DoH / DoQ。
- **HTTP / HTTPS / HTTP/2 / HTTP/3**：當代網路的萬用協定。
- **TLS / mTLS**：加密 + 身分驗證（PKI、ECDHE、AEAD）。
- **電郵**：SMTP / IMAP / POP3，加上 SPF / DKIM / DMARC。
- **檔案 / 媒體**：FTP / SFTP、SMB、NFS、S3、HLS / DASH 串流。
- **即時**：WebRTC（媒體 + 資料 + ICE / STUN / TURN）。

軟體本科這部分都熟，本科視角是「協定設計的 trade-off 與通訊瓶頸」。

## 第 7 章　安全（一窺）

- **TLS** 流程：握手 → 對稱密鑰 → AEAD。
- **PKI**：根憑證 → 簽發憑證 → 信任鏈。
- **網路攻擊**：DDoS、ARP spoofing、DNS 劫持、MITM、Replay、injection。
- **防護**：防火牆、IDS / IPS、SD-WAN、Zero Trust、DNSSEC。

## 第 8 章　無線網路在計網的視角

把實體 / 連結層替換成 Wi-Fi、4G/5G、衛星等：
- Wi-Fi：802.11 MAC（CSMA/CA、RTS/CTS、QoS WMM、漫遊 802.11r）。
- 蜂巢式：核心網（EPC 4G、5GC 5G）以 IP 為主、控制平面與使用者平面分離（CUPS）。
- 整合到網路：行動位址（Mobile IP 概念在 5G 中由 PDU session 處理）、漫遊計費。

## 第 9 章　網路效能與監測

- **吞吐量**：iperf、TCP/UDP 測。
- **延遲**：ping、TWAMP。
- **抖動**、**封包遺失率**：影響 VoIP、串流。
- **擁塞**：rate-based vs window-based。
- **CDN / 邊緣**：把內容拉近用戶，減 RTT。

## 第 10 章　現代演進方向

- **SDN（軟體定義網路）**：控制平面分離、用 OpenFlow / P4。
- **NFV（網路功能虛擬化）**：把專屬硬體換成 x86 / DPU 上的軟體模組。
- **DPU / SmartNIC**：把網路 / 儲存卸載到專用處理器。
- **資料中心網路**：CLOS / leaf-spine、RDMA over Converged Ethernet (RoCE)。
- **5G / 6G + 邊緣運算**：MEC。
- **量子通訊 / QKD**：實驗階段。
