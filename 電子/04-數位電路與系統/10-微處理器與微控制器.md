## 0. 本科在本系的定位

到目前為止，數位電路與系統這條線教的多半是「靜態的硬體」：邏輯閘、組合邏輯、正反器、有限狀態機、用 HDL（Verilog/VHDL）把這些電路描述出來合成成晶片。那是「**怎麼蓋一台機器**」。再往上一層是計算機組織（Computer Organization），講「**機器的內部結構**」——CPU 的資料路徑、控制單元、暫存器檔、流水線、快取怎麼湊成一顆會跑指令的處理器。

這門「微處理器與微控制器」則是把視角拉到**工程師實際拿來用的那顆晶片**。它不再問「ALU 內部怎麼設計」，而是問：

- 我手上這顆晶片，**指令集**長什麼樣？怎麼透過它控制硬體？
- 它怎麼跟外面的世界互動——讀按鈕、點 LED、量電壓、轉馬達、跟感測器講話？
- 它怎麼一邊跑主程式、一邊「被外界事件打斷」去處理急事（中斷）？
- 怎麼精準地數時間、產生波形（計時器/PWM）？

對軟體背景很強的擁有者，最直覺的定位是：**這門課是「軟體」與「真實電子世界」之間的那層介面（the hardware/software boundary）**。你已經很會寫在作業系統與函式庫之上的程式；嵌入式的世界是把那層全部掀開——沒有 OS（或只有極薄的 RTOS）、直接對著「**暫存器位址**」寫值，一個位元就決定一支腳是高電位還是低電位。學會這層，你就能把前面類比電路處理過的訊號、ADC 轉出來的數字，一路接進你熟悉的程式邏輯。

關聯整理（這門課站在三門課的交會口）：

- **數位邏輯 / HDL**：提供「磚塊」。微控制器內部的每個周邊（UART、計時器、SPI）本質上都是一塊用 HDL 設計、被合成進晶片的數位電路（狀態機 + 暫存器）。你用軟體寫的，其實是去設定這些硬體狀態機的「設定暫存器」。
- **計算機組織**：提供「CPU 核心」。本課把核心當成已知的黑盒（取指、解碼、執行），重點放在核心**外面**的記憶體與周邊。
- **類比電路 / 訊號**：提供「外界訊號」。ADC/DAC、運放前端、感測器把連續世界轉成數字，本課負責把這些數字收進處理器。

## 1. 微處理器 vs 微控制器：差在哪

兩者核心都是一顆 CPU，差別在「**整合程度與用途**」。

- **微處理器（Microprocessor, MPU / CPU）**：晶片裡基本上只有運算核心（加上快取）。記憶體（DRAM）、儲存、各種 I/O 控制器都在**晶片外面**，要靠主機板把它們接起來。代表：桌機/伺服器的 x86、手機的高階應用處理器（跑 Linux/Android 的 Cortex-A 系列）。特徵是「算力強、要外掛一堆東西、跑完整作業系統」。
- **微控制器（Microcontroller, MCU）**：把 CPU 核心 + 程式記憶體（Flash）+ 資料記憶體（SRAM）+ 一堆周邊（GPIO、計時器、ADC、UART/SPI/I2C…）**全部塞進同一顆晶片**。買一顆就是一台小電腦。特徵是「便宜、低功耗、即時、單晶片就能控制一個東西」。代表：8051、AVR（Arduino）、PIC、ARM Cortex-M 系列（STM32、NXP、Microchip SAM 等）。

一句話比喻：**微處理器是「一顆超強的腦，但要外接器官」；微控制器是「一隻完整的小昆蟲，腦不大但五臟俱全」**。冰箱、洗衣機、滑鼠、無人機飛控、汽車的每個小模組裡，跑的幾乎都是 MCU 而不是 MPU。這門課的重心在 **MCU**，因為「介面」的故事在 MCU 上最完整。

## 2. 處理器架構與匯流排

### 2.1 三條匯流排（bus）

CPU 要從記憶體拿指令與資料、把結果寫回，靠的是**匯流排**——一束平行的導線，加上控制訊號協調誰在什麼時候講話。傳統上分三組：

- **位址匯流排（address bus）**：CPU 送出「我要存取哪個位址」。寬度決定可定址空間，$N$ 條位址線可定址 $2^N$ 個位置。例如 16 條位址線 → $2^{16}=64\,\text{KB}$。
- **資料匯流排（data bus）**：實際搬運的資料。寬度（8/16/32/64 位元）通常就是大家口中的「幾位元的處理器」。
- **控制匯流排（control bus）**：方向與時序訊號，如讀（RD）、寫（WR）、時脈、中斷請求等。

直覺：位址像「門牌號碼」，資料像「貨物」，控制像「現在是要送貨還是取貨、什麼時間點動作」。

### 2.2 馮諾伊曼 vs 哈佛架構

這是本課最常考、也最能體現「匯流排」概念的對比，差別在於「**程式（指令）和資料是否共用同一條記憶體通道**」。

- **馮諾伊曼架構（Von Neumann）**：指令與資料放在**同一塊記憶體、共用同一條匯流排**。CPU 每個時刻只能做一件事——要嘛取指令、要嘛存取資料，兩者搶同一條路。這個瓶頸俗稱 **馮諾伊曼瓶頸（Von Neumann bottleneck）**。優點是簡單、彈性（程式可被當成資料讀寫，利於載入、自我修改、JIT）。代表：x86、ARM Cortex-A、多數通用處理器。
- **哈佛架構（Harvard）**：指令記憶體與資料記憶體**各有獨立的匯流排**，可以「同時」取下一條指令又存取這次的資料，少了搶路的延遲，吞吐較高，代價是硬體較複雜。代表：8051、AVR、PIC，以及多數 DSP。
- **改良型哈佛（Modified Harvard）**：現實中的折衷。對外像馮諾伊曼（單一統一位址空間，方便程式存取常數表、把 Flash 當資料讀），對內用分離的指令/資料快取或匯流排取得哈佛的速度。**ARM Cortex-M 多屬此類**：它有統一的位址空間（後述記憶體映射），但內部用 AHB/AXI 多匯流排矩陣，能並行取指與取資料。

口訣：**「分流（哈佛）求快，合流（馮諾伊曼）求簡」**；現代高效能 MCU 幾乎都是「外觀合流、內部分流」的改良型哈佛。

### 2.3 RISC vs CISC（順帶的指令集哲學）

- **CISC（複雜指令集）**：指令多、單一指令可做複雜的事（如記憶體到記憶體運算），指令長度不定，硬體解碼複雜。代表 x86。
- **RISC（精簡指令集）**：指令少而規整、定長、多為單週期、只有 load/store 能碰記憶體（load-store 架構）。利於流水線與低功耗。代表 ARM、RISC-V、AVR。**幾乎所有現代 MCU 走 RISC 路線**。

## 3. 指令集概念（ISA）

**指令集架構（Instruction Set Architecture, ISA）** 是硬體與軟體之間的「合約」：它規定了處理器看得懂哪些指令、指令怎麼編碼、有哪些暫存器、記憶體怎麼定址。編譯器產生的機器碼就是針對某個 ISA。對軟體人最關鍵的觀念：**你寫的 C 最終被翻成這些指令，而 ISA 決定了「能對硬體做什麼」的邊界**。

一個典型 ISA 大致包含幾類指令：

- **資料搬移**：`MOV`（暫存器/記憶體間搬資料）、`LDR`/`STR`（load/store，RISC 唯二能碰記憶體的指令）、`PUSH`/`POP`（堆疊）。
- **算術/邏輯**：`ADD`、`SUB`、`MUL`、`AND`、`OR`、`XOR`、`SHIFT`，運算結果會影響**旗標暫存器**（zero、carry、negative、overflow）。
- **流程控制**：`B`/`JMP`（跳躍）、`BEQ`/`BNE`（條件分支，看旗標）、`CALL`/`BL`（呼叫副程式）、`RET`（返回）。
- **特殊/系統**：`NOP`、中斷返回、進入低功耗（`WFI`, wait for interrupt）、原子操作等。

### 3.1 定址模式（addressing modes）

「指令要操作的資料在哪裡」有多種指定方式，這是 ISA 的核心彈性來源：

- **立即定址（immediate）**：運算元就寫在指令裡，如 `MOV R0, #5`。
- **暫存器定址（register）**：運算元在某暫存器，如 `ADD R0, R1, R2`。
- **直接定址（direct/absolute）**：指令給出記憶體絕對位址。
- **間接定址（register indirect）**：暫存器裡放的是「位址」，如 `LDR R0, [R1]`（去 R1 指的位址取值）——這就是 C 的指標。
- **基底加偏移（base + offset / indexed）**：`LDR R0, [R1, #4]`，存取結構體成員、陣列元素的常用法。

### 3.2 ARM 的 Thumb 指令集（為什麼 Cortex-M 省記憶體）

ARM 原生指令是 32 位元定長，但 MCU 上 Flash 很寶貴。**Thumb / Thumb-2** 是一套以 16 位元為主、混入部分 32 位元的壓縮編碼，碼密度（code density）高很多，省 Flash。Cortex-M 系列**只執行 Thumb-2**，不跑傳統 32 位元 ARM 指令，這也是它面積小、適合嵌入式的原因之一。

## 4. 記憶體與 I/O

### 4.1 記憶體種類

MCU 內常見兩大類記憶體：

- **程式記憶體（非揮發，斷電不丟）**：**Flash**（可電氣抹寫，存放韌體）、ROM（出廠燒死）。程式碼與常數放這裡。
- **資料記憶體（揮發，斷電即失）**：**SRAM**（快、放變數、堆疊、堆積）。執行期的所有可變狀態都在這裡。
- 另常見 **EEPROM**（少量、可逐位元組改的非揮發記憶體，存設定值/校正參數）。

哈佛架構的 MCU（如 8051）把程式空間與資料空間在指令層級就分開定址；改良型哈佛（Cortex-M）則把它們映射到**同一個位址空間的不同區段**。

### 4.2 記憶體映射 I/O（memory-mapped I/O）

這是嵌入式最重要、也最該「從基本講清楚」的觀念。

問題：CPU 只會做「讀某位址」「寫某位址」。那它怎麼控制一支實體的腳、或讀一個感測器？

兩種做法：

- **獨立 I/O（port-mapped I/O / isolated I/O）**：周邊有獨立的位址空間，用專門的 `IN`/`OUT` 指令存取（x86 傳統做法、8051 也有此味道）。
- **記憶體映射 I/O（memory-mapped I/O, MMIO）**：把每個周邊的**控制/狀態/資料暫存器**直接「貼」到一般記憶體位址空間的某些位址上。**對那個位址寫值 = 設定硬體；讀那個位址 = 讀硬體狀態**。ARM Cortex-M 全面採用 MMIO。

直覺與威力：在 Cortex-M 上，要把 GPIO Port A 的第 5 腳設成高電位，本質就是：

$$\text{*(volatile uint32\_t *)(GPIOA\_BASE + ODR\_OFFSET)} \;|=\; (1 \ll 5);$$

也就是「往一個固定位址寫一個位元」。**這正是軟體與硬體的接點**：暫存器在硬體上是一排正反器（由 HDL 合成的數位電路），MMIO 讓它「看起來」就像記憶體裡的一個變數。注意 C 裡一定要加 `volatile`，告訴編譯器「這個位址的值會被硬體改變、別把讀寫最佳化掉」。

Cortex-M 還有個貼心設計：位址空間被切成固定區段（程式碼區、SRAM 區、周邊區、私有周邊區 PPB 等），所以同系列晶片的記憶體佈局可預期，CMSIS 標準庫據此提供統一的暫存器定義。

### 4.3 DMA（直接記憶體存取）

**DMA（Direct Memory Access）** 是一個能「不勞煩 CPU 就在記憶體與周邊間搬資料」的硬體引擎。例如 ADC 連續取樣 1000 點，若每點都中斷 CPU 來搬會很累；改用 DMA，CPU 設定好「從 ADC 資料暫存器搬 1000 筆到 SRAM 這塊緩衝區」後就放手去做別的事，搬完才中斷通知。**DMA 是高吞吐周邊（ADC、SPI、UART）不拖垮 CPU 的關鍵**。

## 5. 中斷（Interrupts）

### 5.1 為什麼要中斷：輪詢 vs 中斷

CPU 怎麼知道「按鈕被按了」「資料到了」？兩種策略：

- **輪詢（polling）**：CPU 在迴圈裡反覆讀狀態旗標問「好了沒？好了沒？」。簡單，但浪費 CPU、反應慢、難兼顧多事件。
- **中斷（interrupt）**：平時 CPU 做主程式，事件發生時硬體**主動打斷** CPU，強迫它跳去執行對應的**中斷服務常式（ISR, Interrupt Service Routine）**，做完再回到原處。

對軟體人最好的比喻：**輪詢像「不停 F5 重新整理收件匣」，中斷像「來信時手機跳通知」**。嵌入式系統的即時性幾乎全靠中斷。

### 5.2 中斷處理的硬體流程

事件發生 → CPU 在當前指令邊界停下 → 自動把當前狀態（PC、旗標等）**存到堆疊**（保存現場）→ 透過**中斷向量表（vector table）** 查到該事件對應的 ISR 入口位址 → 跳去執行 ISR → ISR 結束時把現場從堆疊還原（在 Cortex-M 上由特殊返回值觸發 exception return）→ 回到被打斷的地方繼續。

幾個關鍵概念：

- **中斷向量表**：一張位址表，每個中斷源對應一個 ISR 入口位址。Cortex-M 把它放在記憶體最前面，第 0 項是初始堆疊指標、第 1 項是 Reset 入口。
- **優先權（priority）與巢狀（nesting）**：多個中斷可設不同優先權；高優先權中斷可以打斷正在執行的低優先權 ISR，做完再回來——這叫**巢狀中斷**。
- **中斷延遲（interrupt latency）**：從事件發生到 ISR 第一條指令執行之間的時間。即時系統很在意這個數字。
- **可遮罩 vs 不可遮罩**：一般中斷可被軟體「遮罩」（暫時關閉）；NMI（不可遮罩中斷）用於致命事件，關不掉。

### 5.3 Cortex-M 的 NVIC

ARM Cortex-M 把中斷管理做成核心內建的標準單元 **NVIC（Nested Vectored Interrupt Controller，巢狀向量中斷控制器）**，這是 Cortex-M 介面好用的一大原因。特點：

- 「**Vectored**」：向量化——硬體直接從向量表查到 ISR 位址跳過去，不必像舊架構那樣先進一個總入口再用軟體判斷是誰，省時間。
- 「**Nested**」：支援可程式化的優先權與巢狀。
- 自動保存/還原部分暫存器（硬體 stacking），讓 ISR 可以直接用 C 寫，不必手動存暫存器。
- 支援 **tail-chaining**（連續中斷時省去多餘的還原-再保存）等最佳化，降低延遲。
- 異常編號：前 16 號為系統異常（Reset、NMI、HardFault、SVCall、SysTick 等），16 號之後是廠商定義的周邊中斷（外部腳、計時器、UART…）。

此外 Cortex-M 內建一個 **SysTick**——24 位元的系統計時器，常用來產生固定節拍（如 RTOS 的排程 tick 或 1ms 心跳）。

## 6. 計時器與 PWM（Timer / Counter / PWM）

### 6.1 計時器/計數器的本質

**計時器（timer）本質就是一個會自動加一的計數暫存器**。它接一個已知頻率的時脈，每來一個時脈邊緣計數值加一；數到頂（溢位）就觸發事件/中斷並歸零。當時脈來源是內部固定頻率，它在「**計時**」；當時脈來源是外部腳的脈衝，它在「**計數**」外部事件。

為了能數較長的時間，前面常接一個**預除頻器（prescaler）**：把輸入時脈先除以 $N$ 再餵給計數器。設系統時脈 $f_{clk}$、預除值 $P$、計數上限（自動重載值）$ARR$，則溢位（一個週期）的時間為：

$$T = \frac{(P)\,(ARR+1)}{f_{clk}}$$

例：$f_{clk}=72\,\text{MHz}$，要產生 1 ms 中斷，可取 $P=72$（得到 1 MHz、即每 µs 計一次），$ARR+1=1000$，則 $T=1\,\text{ms}$。這種「分頻 + 計數到頂中斷」是嵌入式產生精準節拍的標準手法。

計時器常見模式：**週期中斷**（定時做事）、**輸入捕捉（input capture）**（記錄外部訊號邊緣到來的時刻，用來量脈寬、量頻率、量轉速）、**輸出比較（output compare）**（計數值到某數時翻轉輸出腳）。

### 6.2 PWM（脈寬調變）

**PWM（Pulse-Width Modulation，脈寬調變）** 是用計時器產生「固定週期、可調工作週期（duty cycle）」的方波。**工作週期 $D$ = 高電位時間佔整個週期的比例**：

$$D = \frac{t_{high}}{T} \times 100\%$$

實作上：計時器從 0 數到 $ARR$ 為一個 PWM 週期（決定頻率），另設一個比較值 $CCR$；當計數值 $< CCR$ 輸出高、否則輸出低，於是 $D = CCR/(ARR+1)$。改 $CCR$ 就改 duty。

為什麼 PWM 這麼有用：很多「類比效果」可以用「快速切換的數位訊號 + 慣性/低通濾波」近似——

- **調光/調速**：LED 亮度、直流馬達轉速看的是「平均功率」，平均電壓 $\approx D \times V_{supply}$，改 duty 就改亮度/轉速，且開關元件全程要嘛全開要嘛全關，**效率遠高於線性調節**（不發熱在電阻上）。
- **馬達/伺服控制**：三相 PWM 驅動 BLDC/步進馬達；RC 伺服機用固定週期、特定脈寬編碼角度。
- **電源轉換**：開關式電源（buck/boost）的核心就是高頻 PWM。
- **粗糙的 DAC**：PWM 經 RC 低通可得到正比於 duty 的類比電壓。

## 7. ADC 與 DAC（連續世界的入口/出口）

真實世界（溫度、光、聲音、電壓）是**連續類比量**，CPU 只懂離散數字，必須轉換。

### 7.1 ADC（類比數位轉換器）

**ADC（Analog-to-Digital Converter）** 把類比電壓量化成數字。三個核心參數：

- **解析度（resolution）**：幾位元。$N$ 位元把參考電壓 $V_{ref}$ 切成 $2^N$ 階。每階大小（LSB）為：

$$\text{LSB} = \frac{V_{ref}}{2^N}$$

換算公式：輸入電壓 $V_{in}$ 轉成數字碼 $D = \mathrm{round}\!\left(\dfrac{V_{in}}{V_{ref}}\,(2^N-1)\right)$。例：12 位元、$V_{ref}=3.3\,\text{V}$，則最小可分辨約 $3.3/4096 \approx 0.8\,\text{mV}$。

- **取樣率（sampling rate）**：每秒取幾個點。由**奈奎斯特定理（Nyquist）**，取樣率須大於訊號最高頻率的兩倍才能無失真重建，否則發生**混疊（aliasing）**——所以 ADC 前常加**抗混疊低通濾波器**（與類比電路那門課接軌）。
- **常見架構**：MCU 內建的多為**逐次逼近型（SAR, Successive Approximation）**，速度與精度折衷得好；高速用 flash/pipeline ADC；高精度低速用 Σ-Δ（sigma-delta）。

實務上 ADC 常配 DMA：設定好取樣序列與緩衝區，硬體連續取樣、自動搬進 SRAM，CPU 只在整批完成時被中斷一次。

### 7.2 DAC（數位類比轉換器）

**DAC（Digital-to-Analog Converter）** 反過來，把數字碼轉成類比電壓輸出，用於產生音訊、波形、控制電壓。許多低階 MCU 沒有真正的 DAC，就用「PWM + 低通濾波」近似。

## 8. GPIO（通用輸入輸出腳）

**GPIO（General-Purpose Input/Output）** 是 MCU 最基本的對外介面：一支可由軟體控制的數位腳，能當輸出（吐高/低電位）或輸入（讀外界高/低電位）。透過前述 MMIO，每支腳的行為由幾個暫存器決定（名稱依廠商，概念相通）：

- **方向暫存器（direction/mode）**：設這支腳是輸入還是輸出。
- **輸出資料暫存器（output data, 如 ODR）**：當輸出時，寫 1/0 決定高/低電位。
- **輸入資料暫存器（input data, 如 IDR）**：當輸入時，讀它得到外界電位。
- **置位/清除暫存器（set/reset, 如 BSRR）**：原子地把某腳設 1 或清 0，避免「讀-改-寫」過程被中斷打斷的競態。
- **上拉/下拉（pull-up/pull-down）**：腳沒接東西時若浮接（floating），讀值會亂跳；內建上拉/下拉電阻把它釘在已知電位（按鈕電路最常用）。
- **替代功能（alternate function）**：同一支實體腳常可被切換去接 UART/SPI/I2C/計時器等周邊，這叫**腳位多工（pin muxing）**。所以晶片資料表會列每支腳的多種可能用途。

輸出驅動型態也要懂：**推挽（push-pull）** 能主動拉高也拉低；**開漏（open-drain）** 只能拉低、拉高靠外部上拉電阻——I2C 的 SDA/SCL 就是開漏，才允許多個裝置共線且做線及邏輯。

讀按鈕還有個經典坑：**機械開關彈跳（bounce）**——按一下實際產生多次抖動的高低變化。解法是**去彈跳（debounce）**：軟體延遲再確認、或硬體 RC + 施密特觸發。

## 9. 串列通訊（Serial Communication）

晶片之間要互相講話。**並列傳輸**（一次送多位元）快但耗線；**串列傳輸**（一次送一位元、靠時間排隊）省線，是板內/板間通訊主流。三大常見協定 UART、SPI、I2C 各有取捨，這是本課的重頭戲。

### 9.1 同步 vs 非同步

- **同步（synchronous）**：另有一條**時脈線**告訴接收方「現在這一刻取樣這個位元」。雙方靠共同時脈對齊，可跑很快。SPI、I2C 屬此。
- **非同步（asynchronous）**：**沒有共用時脈線**，雙方事先約好速率（鮑率），各用自己的時脈，靠資料格式裡的起始位元對齊。UART 屬此。

### 9.2 UART（通用非同步收發器）

**UART（Universal Asynchronous Receiver/Transmitter）** 是點對點、全雙工、非同步的兩線通訊：

- 接線：**TX**（傳）、**RX**（收），交叉相接（A 的 TX 接 B 的 RX）。沒有時脈線。
- 速率叫**鮑率（baud rate）**，常見 9600、115200 bps，雙方必須設成一樣。
- 一個字元的框架（frame）：線路閒置時為高電位 →**起始位元（start bit，拉低 1 位元）** → 5~9 個資料位元（通常 8，LSB 先送）→ 選用的**同位元（parity，奇/偶校驗）** → **停止位元（stop bit，高電位 1~2 位元）**。接收端偵測到起始位元的下降緣後，依約定鮑率在每個位元的中央取樣。
- 因為沒共用時脈，雙方時脈誤差須夠小（一般容許幾個百分點），否則到字元尾端取樣點會偏掉。
- 只能一對一。電腦上的「序列埠 / COM port」、模組除錯訊息、GPS、藍牙模組指令，多走 UART。實體層常再配 RS-232、RS-485、USB-轉序列等。

### 9.3 SPI（串列周邊介面）

**SPI（Serial Peripheral Interface）** 是同步、全雙工、主從式的高速匯流排，典型四線：

- **SCLK**：主機產生的時脈。
- **MOSI**（Master Out, Slave In）：主→從資料。
- **MISO**（Master In, Slave Out）：從→主資料。
- **SS / CS**（Slave Select / Chip Select，低電位有效）：主機選擇要跟哪個從機講話；每多一個從機就多一條 CS 線。

運作直覺：主從各有一個移位暫存器接成一個環，每個 SCLK 邊緣同時「主送一位、從送一位」，$n$ 個時脈後雙方各自的位元剛好換完——所以 SPI 天生全雙工。沒有位址、沒有確認（ACK）、框架很薄，所以**很快**（數十 MHz 等級），適合 SD 卡、Flash、顯示器、高速 ADC。代價：線多（尤其多從機時 CS 線一堆）、傳輸距離短、沒有內建確認機制。

時脈極性/相位（CPOL/CPHA）構成四種 **SPI 模式**，規定在時脈的哪個邊緣取樣，主從必須設一致，否則對不上。

### 9.4 I2C（積體電路間匯流排）

**I2C（Inter-Integrated Circuit，也寫 IIC / I²C）** 是同步、半雙工、多主多從的**兩線**匯流排，主打「省腳、掛很多裝置」：

- **SDA**（資料）、**SCL**（時脈），兩條線全部裝置**共用**。兩線都是**開漏 + 外部上拉電阻**，平時被拉高，誰要拉低就拉低（線及邏輯 wired-AND）。
- **用位址定址**：每個從機有 7 位元（或 10 位元）位址，主機在傳輸開頭廣播位址，被叫到的從機才回應——所以同一對線上可掛幾十個裝置而不用額外選擇線。
- 框架：**起始條件（START：SCL 高時 SDA 由高轉低）** → 7 位元位址 + 1 位元讀寫方向 → 從機回 **ACK/NACK** → 資料位元組（每組後都有 ACK）→ **停止條件（STOP：SCL 高時 SDA 由低轉高）**。每個位元組後的 ACK 提供了「對方有沒有收到」的確認，這是 SPI 沒有的。
- 速度分級：標準 100 kHz、快速 400 kHz、更高的 1 MHz/3.4 MHz 等。因為框架開銷（位址、ACK、起停）較多，同樣時脈下**比 SPI 慢**。
- 適合：感測器（溫濕度、IMU、氣壓）、RTC、小型 EEPROM、I/O 擴充器——「慢速、省線、多裝置」的場景。

### 9.5 三者一句話總結

| 協定 | 同步性 | 線數 | 拓樸 | 速度 | 確認機制 | 典型用途 |
|---|---|---|---|---|---|---|
| **UART** | 非同步 | 2（TX/RX） | 點對點 | 中（~115k bps 常見） | 無（可加 parity） | 除錯、模組指令、序列埠 |
| **SPI** | 同步 | 4＋（每從機多一 CS） | 一主多從 | 高（數十 MHz） | 無 | SD/Flash、顯示器、高速感測 |
| **I2C** | 同步 | 2（SDA/SCL，共用） | 多主多從、位址定址 | 低中（100k~400k 常見） | 有（ACK） | 慢速感測器、RTC、EEPROM |

選型直覺：**要快、線可以多 → SPI；要省線掛很多裝置 → I2C；對外接電腦/模組、點對點 → UART。**

## 10. 典型微控制器：8051 與 ARM Cortex-M

### 10.1 8051（經典 8 位元教學機）

1980 年代 Intel 推出、至今仍是教學與低階應用的標竿，是**理解 MCU 結構的最佳入門模型**。

- **架構**：8 位元、**哈佛架構**（程式與資料記憶體分離定址）、CISC 風格。
- **記憶體**：經典版內部 4 KB 程式 ROM、128 bytes 內部 RAM；可外接最多 64 KB 程式 + 64 KB 資料記憶體。內部 RAM 的低 32 bytes 分成 4 組各 8 個的**暫存器庫（register banks，R0–R7）**。
- **SFR（特殊功能暫存器，Special Function Registers）**：位於 0x80–0xFF 的一塊位址區，約 21 個，是**8051 版的「記憶體映射 I/O」**——埠、計時器、串列埠、中斷的控制全靠讀寫這些 SFR。重要的如：累加器 `ACC`、`B`、程式狀態字 `PSW`（含旗標）、堆疊指標 `SP`、埠 `P0~P3`、計時器 `TMOD/TCON/THx/TLx`、串列 `SCON/SBUF`、中斷 `IE/IP`。
- **I/O 埠**：4 個 8 位元埠 P0–P3（共 32 支腳）。P0 在外接記憶體時兼作**多工的位址/資料匯流排**（位址低位元組與資料分時共用同一組腳，需外接鎖存器分離）。
- **計時器**：兩個 16 位元計時器/計數器（Timer 0、Timer 1），可定時或計外部脈衝，常被拿來產生 UART 鮑率。
- **中斷**：原始版 5 個中斷源（2 外部、2 計時器、1 串列），用 `IE` 開關（`EA` 為總開關）、`IP` 設兩級優先權。
- **串列**：內建一個 UART。

8051 的價值在於：**小到可以把整張暫存器表記在腦裡，看清「軟體寫 SFR ↔ 硬體動作」的因果鏈**。

### 10.2 ARM Cortex-M（現代 32 位元主流）

當代嵌入式的事實標準，STM32、NXP LPC、Microchip SAM、樹莓派 Pico（RP2040）等都基於它。子型號依算力與功能分層：

- **Cortex-M0 / M0+**：最小、最省電，取代 8 位元 MCU 的入門款。
- **Cortex-M3 / M4**：M4 多了 DSP 指令與選配的單精度浮點（FPU），適合訊號處理、馬達控制。
- **Cortex-M7**：高效能、有快取與流水線，做進階控制與運算。
- **Cortex-M33 等**：加入 TrustZone 安全延伸。

共通特徵（也是它好用的原因）：

- **32 位元 RISC、改良型哈佛、只跑 Thumb-2**：碼密度高、效能/面積/功耗平衡好。
- **統一的記憶體映射 + MMIO**：固定的位址區段佈局，配合 **CMSIS**（ARM 提供的標準軟體層）讓不同廠商晶片的暫存器存取方式一致。
- **內建 NVIC**：標準化、向量化、可巢狀的中斷控制，ISR 可直接用 C 寫。
- **內建 SysTick**：標準系統節拍計時器，方便接 RTOS。
- **豐富的廠商周邊**：GPIO、多組計時器/PWM、多通道 ADC（常含 DAC）、多組 UART/SPI/I2C、CAN/USB/乙太網路、DMA 等，全部以 MMIO 暴露。
- **低功耗模式**：`WFI`/`WFE` 指令搭配多級睡眠，電池供電的關鍵。

### 10.3 開發流程（軟體角度，精簡）

對軟體能力強的擁有者，這段可以很快帶過：交叉編譯器（如 GCC for ARM）把 C/組語編成韌體 → 連結腳本（linker script）依晶片記憶體佈局安排程式進 Flash、變數進 SRAM、設好向量表 → 透過 **JTAG/SWD** 介面用偵錯器把映像燒進 Flash 並單步偵錯。常見抽象層次：直接寫暫存器（最底、最透明）→ 廠商 HAL/LL 庫 → RTOS（如 FreeRTOS）→ 上層應用。**越往上越像你熟悉的軟體開發，越往下越貼近本課講的硬體暫存器**。

## 11. 一個貫穿全課的範例（把概念串起來）

想像做一個「**室溫風扇控制器**」，它把本課所有零件都用上：

1. **ADC** 連續量測溫度感測器電壓 → 透過 **DMA** 自動搬進 SRAM 緩衝區，CPU 不被頻繁打擾。
2. **計時器** 每 100 ms 產生一次**中斷**，ISR 裡讀最新溫度、跑控制邏輯，算出風扇該轉多快。
3. 用 **PWM**（另一組計時器的輸出比較）調整風扇馬達的工作週期，溫度越高 duty 越大。
4. 面板上一顆**按鈕**接 **GPIO 輸入**（含上拉與去彈跳），按一下切換自動/手動模式。
5. 狀態 LED 接 **GPIO 輸出**。
6. 透過 **I2C** 讀一顆數位溫濕度感測器與 RTC（記錄時間）。
7. 透過 **UART** 把運轉紀錄吐到電腦終端機除錯。
8. 整個系統可選擇直接寫暫存器，或在 **Cortex-M + FreeRTOS** 上把「讀感測器」「控風扇」「印 log」拆成不同任務。

這就是嵌入式系統的縮影：**CPU 核心當大腦，周邊當手腳與感官，中斷與計時器當神經反射，串列介面當對外的嘴巴與耳朵，全部透過記憶體映射的暫存器由軟體統一指揮。**

## 12. 碩士層級延伸（簡短）

- **即時作業系統（RTOS）與排程**：任務排程、優先權反轉、可預測延遲、硬即時 vs 軟即時。
- **多核與非對稱架構（AMP/SMP）**：MCU 上的雙核（如 M0+ 配 M4）、核間通訊。
- **低功耗與能量收集設計**：超低功耗 MCU、duty-cycling、能量收集（energy harvesting）供電的系統。
- **嵌入式安全（security）**：TrustZone、安全開機（secure boot）、加密加速器、側通道攻擊防護。
- **功能安全（functional safety）**：汽車 ISO 26262、工業 IEC 61508，冗餘與容錯設計。
- **進階匯流排與互連**：AMBA（AHB/APB/AXI）匯流排矩陣、片上網路（NoC）。
- **DSP 與機器學習於 MCU（TinyML）**：Cortex-M 上跑量化神經網路、CMSIS-NN、邊緣推論。
- **高階綜合與軟硬協同設計**：把部分演算法落到 FPGA/加速器，與 MCU 協同。
- **進階通訊匯流排**：CAN/CAN-FD、LIN、USB、乙太網路（含 TSN）、無線（BLE、Zigbee、LoRa）的協定堆疊。

## 13. 博士層級研究方向（列表）

- 超低功耗與近閾值（near-threshold）運算的處理器微架構。
- 非揮發處理器（non-volatile / intermittent computing）與能量收集系統的可間斷執行模型。
- 邊緣 AI 加速器與記憶體內運算（in-memory / compute-in-memory）的 MCU 整合。
- 即時系統的最壞情況執行時間（WCET）分析與可預測微架構。
- 嵌入式系統的硬體安全：側通道與故障注入攻擊的偵測與防護、PUF、根信任。
- RISC-V 開放指令集在客製化嵌入式處理器與領域專用擴充（DSA）上的應用。
- 混合訊號 SoC 整合：ADC/DAC、感測器前端與數位核心的協同設計與校正。
- 容錯與功能安全架構：輻射環境（太空/汽車）下的冗餘與錯誤偵測機制。
- 軟硬協同的形式化驗證：周邊驅動與中斷處理的正確性保證。
- 自適應/近似運算（approximate computing）於能耗受限嵌入式平台。
