# 計算機組織（Computer Organization）

> 名詞對照見 [../中英名詞對照表.md](../中英名詞對照表.md)

本科目是電子工程「數位電路與系統」階段的頂點：把前面學過的**數位邏輯**（閘、正反器、暫存器、有限狀態機、加法器/ALU）組裝成一台**會執行程式的處理器**，並把處理器擴張成一個包含**記憶體階層、I/O 與匯流排**的完整計算機。對應教材主要是 Patterson & Hennessy 的《Computer Organization and Design》（俗稱 P&H、COD），課程以 MIPS（或 RISC-V）為示範架構。

對擁有者（電機/通訊背景、軟體強）特別說明定位：
- 你寫過很多程式，知道 `add`、`load`、迴圈、函式呼叫長什麼樣。**計算機組織就是回答「這些指令在硬體上到底怎麼被一拍一拍跑出來」**——把抽象的指令對應到實體的線路、暫存器與時脈邊緣。
- 因此本篇刻意**從硬體直覺切入**：每個概念都問「哪些線、哪個正反器、第幾個時脈被誰寫進去」。純軟體面（組譯、編譯、OS 分頁政策的細節）只點到為止。

## 0. 兩個分層：ISA、組織、實作

要先把三個常被混用的詞分清楚，後面才不會亂：

- **指令集架構（ISA, Instruction Set Architecture）**：軟體看得到的「介面契約」——有哪些指令、有幾個暫存器、定址方式、資料型別。這是程式設計師/編譯器面對的抽象。例：MIPS、RISC-V、x86、ARM 都是 ISA。
- **計算機組織（Organization / Microarchitecture）**：在硬體上**如何實現**這個 ISA——要不要管線、快取多大、有幾個 ALU。同一個 ISA 可以有很多種組織（例如同樣是 x86，Intel 與 AMD 的微架構不同）。**這科講的就是這一層。**
- **實作（Implementation）**：用什麼製程、邏輯閘、佈局把組織做成晶片（這屬於 VLSI/數位 IC）。

直覺：ISA 是「規格書」，組織是「工程設計圖」，實作是「真的蓋出來」。

### 0.1 預備：把 ISA 當成資料路徑的「需求清單」

以 MIPS 為例，指令分三種格式：
- **R 型**（暫存器運算）：`add $rd, $rs, $rt`——讀兩個暫存器、ALU 運算、寫回一個暫存器。
- **I 型**（含立即值/載入儲存/分支）：`lw $rt, offset($rs)`、`beq $rs, $rt, label`——含 16 位元立即值。
- **J 型**（跳躍）：`j target`。

我們的目標：**設計一套線路（資料路徑）能跑完這幾類指令**，再設計一個**控制單元**告訴這些線路每一步該怎麼動。

## 1. 資料路徑（Datapath）與控制單元（Control Unit）

任何處理器都可拆成兩部分，這個切分是整科的骨架：

- **資料路徑（Datapath）**：負責「搬資料、算資料」的元件與連線——PC、指令記憶體、暫存器檔（register file）、ALU、資料記憶體、加法器、多工器（MUX）。它本身**不知道現在在做什麼**，只是一堆可被指揮的功能單元。
- **控制單元（Control Unit）**：看著當前指令的 opcode（與 function code），產生一堆**控制訊號**（MUX 選哪一路、暫存器檔要不要寫、記憶體要讀還是寫、ALU 做哪個運算），指揮資料路徑。

**直覺類比**：資料路徑是樂團（各種樂器/功能單元），控制單元是指揮（決定誰在第幾拍出聲）。

### 1.1 指令執行的五個通用步驟

幾乎所有指令都能用這五階段描述（這也是後面管線分段的依據）：

1. **IF（Instruction Fetch）取指**：用 PC 當位址，從指令記憶體讀出指令；同時 `PC ← PC + 4`。
2. **ID（Instruction Decode / Register read）解碼與讀暫存器**：解出 opcode/欄位，從暫存器檔讀出 `$rs`、`$rt`。
3. **EX（Execute）執行**：ALU 做運算（算術、或算記憶體位址、或比較）。
4. **MEM（Memory access）存取記憶體**：`lw` 讀、`sw` 寫；其它指令此階段閒置。
5. **WB（Write Back）寫回**：把結果寫回暫存器檔。

### 1.2 關鍵元件回顧（與數位邏輯的接點）

- **暫存器檔（Register file）**：一堆暫存器 + 兩個讀埠 + 一個寫埠。讀是**組合邏輯**（給編號就出資料）；寫由時脈邊緣 + 寫致能（RegWrite）控制——這正是數位邏輯的「邊緣觸發正反器陣列 + 解碼器」。
- **ALU**：做加減、AND、OR、SLT（小於則設定）等。控制訊號 `ALUop` 決定運算；它就是數位邏輯課做的那顆加法器/邏輯單元。
- **多工器（MUX）**：在「兩個來源」中選一個，由控制訊號當選擇線。資料路徑裡到處是 MUX，因為不同指令要把不同來源接到同一個元件（例如 ALU 第二輸入：R 型用 `$rt`，I 型用立即值，靠 `ALUSrc` 這條 MUX 選擇線決定）。
- **符號延伸（Sign-extend）**：把 16 位元立即值補成 32 位元。

### 1.3 控制訊號舉例（單週期 MIPS）

控制單元基本上是一張**真值表 / 組合邏輯**，輸入 opcode、輸出這些訊號：

- `RegDst`：寫回暫存器編號取 `$rt`（I 型）還是 `$rd`（R 型）。
- `ALUSrc`：ALU 第二輸入取暫存器還是立即值。
- `MemRead` / `MemWrite`：資料記憶體讀/寫。
- `MemtoReg`：寫回的資料來自 ALU 還是記憶體。
- `RegWrite`：是否寫回暫存器。
- `Branch`：是否為分支（配合 ALU 的 Zero 旗標決定是否跳）。
- `ALUOp`：交給「ALU 控制」次級邏輯，結合 function code 決定實際運算。

這就是把 opcode 對應到控制位元的一張表——本質是數位邏輯的組合電路設計。

## 2. 單週期 vs. 多週期實作

### 2.1 單週期（Single-cycle）

**設計**：每條指令在**一個時脈週期內**完整跑完上述五步。控制單元是純組合邏輯（看 opcode 直接吐訊號），資料在一個週期裡從 PC 一路流到寫回。

**致命缺點——時脈被最慢指令綁架**：時脈週期必須長到能容納**最長路徑**的指令。在 MIPS，最長的是 `lw`（要走完 IF→ID→EX→MEM→WB 五階段全部）。即使是只需三階段的 `add`，也得等同樣長的週期。

$$
T_{clk} \ge t_{IF} + t_{ID} + t_{EX} + t_{MEM} + t_{WB}
$$

結果：時脈很慢、硬體利用率低（很多週期裡記憶體或 ALU 閒置）。優點只有**控制簡單**。教學上拿來建立直覺，真實處理器不用。

### 2.2 多週期（Multicycle）

**設計**：把一條指令**拆成多個短週期**，每個短週期只做一個步驟（IF、ID、EX、MEM、WB 各佔一個時脈）。週期長度只需容納**最慢的單一步驟**（通常是一次記憶體存取或一次 ALU 運算），所以時脈可以快很多。

**好處**：
- **時脈快**：週期 = 最慢的一個階段，而非整條指令。
- **指令長短不一**：`add` 用 4 個短週期就好，`lw` 用 5 個——快的指令不必陪跑。
- **硬體可重用**：一條指令的不同階段在不同短週期使用同一個單元，因此可以**只用一個 ALU、一個記憶體**（單週期版常需要分開的指令/資料記憶體與額外加法器）。代價是要加**暫存器**（如 IR 指令暫存器、A/B 暫存器、ALUOut）暫存階段間的中間值。

**控制變複雜**：因為一條指令跨多個週期，控制單元不再是純組合邏輯，而是一個**有限狀態機（FSM）**——每個狀態代表「現在在第幾步」，依狀態輸出控制訊號並決定下一狀態。這正好接回數位邏輯課的 FSM 設計。（更複雜的可用**微程式控制 microprogramming**：把控制訊號存成「微指令」表，用微 PC 逐條讀出。）

**比較直覺**：單週期像「不管事情大小，每件事都給一整天」；多週期像「把一天切成小時，事情用幾小時就用幾小時」。但多週期同一時間仍只跑一條指令——要再快，得讓多條指令**重疊**，這就是管線。

## 3. 指令管線（Pipelining）與危障（Hazards）

### 3.1 核心直覺：洗衣店流水線

管線的比喻：一家洗衣店有洗、烘、摺、收四道工序。若一批衣服做完四道才開始下一批（多週期式），很慢。**流水線**做法：第一批在烘的時候，第二批就開始洗——四台機器同時忙。throughput（吞吐量）大增，雖然單一批的**延遲（latency）沒變**。

處理器照搬：把五階段 IF/ID/EX/MEM/WB 各配一段硬體，**同時容納五條不同指令**，每條在不同階段。每個時脈推進一格。

- 理想情況下，**每個時脈完成一條指令**（CPI → 1），相較單一指令需 5 拍，吞吐量約提升到 5 倍（受限於最慢階段與啟動/排空）。
- 階段之間要插**管線暫存器**（IF/ID、ID/EX、EX/MEM、MEM/WB），把上一段算出的中間值與控制訊號傳給下一段。

理想加速比 $\approx$ 管線級數，但實務上達不到，原因就是**危障（hazards）**——讓管線無法每拍順利推進的狀況。分三類：

### 3.2 結構危障（Structural Hazard）

兩條指令在同一拍要用**同一個硬體資源**而硬體不夠。例：若指令與資料共用單一記憶體，IF（取指）與 MEM（存取資料）會在同一拍撞車。
**解法**：複製資源——分開的指令快取與資料快取（哈佛式），暫存器檔同時支援讀與寫。良好設計的 MIPS 管線基本沒有結構危障。

### 3.3 資料危障（Data Hazard）

後面的指令要用到前面指令**還沒寫回**的結果。例：
```
add $s0, $t0, $t1   # $s0 在 WB（第5階段）才寫好
sub $t2, $s0, $t3   # 下一拍就在 EX 要用 $s0 → 還沒寫好！
```
解法有三層：
- **轉送 / 旁路（Forwarding / Bypassing）**：結果其實在 EX 結束（EX/MEM 暫存器）就算出來了，不必等寫回。加一條**旁路線**把 ALU 輸出直接送回下一條的 ALU 輸入。靠**轉送單元**比對暫存器編號決定要不要旁路。這能消除大多數資料危障，**不需停頓**。
- **載入–使用危障（Load-use hazard）**：`lw` 的資料要到 MEM 階段結束才有，若**緊接**的指令就要用，連旁路都來不及，**必須停頓一拍（stall / bubble）**。靠**危障偵測單元**偵測「上一條是 lw 且目的暫存器被下一條讀」時插入氣泡。編譯器也可重排指令把無關指令塞進這個空檔來避免停頓。
- **停頓（Stall / Bubble）**：擋住前段、塞入 NOP（氣泡），等資料就緒。是最後手段，會掉效能。

### 3.4 控制危障（Control Hazard / Branch Hazard）

分支指令（`beq`）要到較後階段才知道**到底跳不跳、跳去哪**，但管線早已開始抓後面的指令——可能抓錯。
解法：
- **停頓**：等分支結果出來再抓（簡單但慢）。
- **提早判斷**：把分支比較搬到 ID 階段，縮短「猜錯」的代價（branch delay 變短）。
- **分支預測（Branch Prediction）**：先猜（靜態如「永遠不跳」、或動態用**分支歷史表/2-bit 飽和計數器**記錄過去行為），猜對就零成本，猜錯才把錯抓的指令沖掉（flush）並罰幾拍。現代處理器靠高準度動態預測器把控制危障降到很低。
- **延遲分支槽（Delay slot）**：MIPS 經典做法——分支後面那一格一定會執行，由編譯器塞有用指令進去。

### 3.5 例外與中斷對管線的衝擊

管線中若某指令發生例外（如未定義指令、溢位），要能**精確**處理：在出錯指令之後進入管線的指令必須被沖掉、之前的要完成，且能記下正確的回復點（精確例外, precise exception）。這讓控制更複雜，是管線設計的難點之一。

> 進階管線（碩士級）：**超純量（superscalar）**每拍發多條指令、**亂序執行（out-of-order）**用記分板/Tomasulo 動態調度、**深管線**拉高時脈——都是在「榨出更多指令級平行（ILP）」，本科只需知道概念。

## 4. 效能評估（Performance）：CPI 與 Amdahl 定律

### 4.1 基本效能方程式

衡量一支程式跑多快，用**執行時間（CPU time）**，而非看時脈或 MIPS 數字（那些會騙人）。核心公式：

$$
\text{CPU time} = \text{指令數}\;(IC) \times \text{CPI} \times \text{時脈週期}\;(T_{clk})
                = \frac{IC \times CPI}{f_{clk}}
$$

- **IC（Instruction Count）**：執行的指令數——受 ISA、編譯器影響。
- **CPI（Cycles Per Instruction）**：平均每條指令幾個時脈——受微架構（管線、危障、快取）影響。
- **$T_{clk}$**：時脈週期——受製程與電路影響。

三者**互相牽制**：例如 RISC 用更多但更簡單的指令（IC↑、CPI↓、$T_{clk}$↓），CISC 反之。只看其中一項（如時脈頻率）比較效能會誤導。

**加權 CPI**：不同類指令 CPI 不同，
$$
CPI = \frac{\sum_i (CPI_i \times IC_i)}{IC} = \sum_i CPI_i \times f_i
$$
其中 $f_i$ 是該類指令的比例。

- 理想單週期：CPI = 1，但 $T_{clk}$ 很大。
- 理想管線：CPI 趨近 1 且 $T_{clk}$ 小——但實際 CPI 會被危障與**快取失誤**拉高（見第 5 節）。

### 4.2 Amdahl 定律（Amdahl's Law）

回答：「只把一部分變快，整體能快多少？」設某項改善只能加速程式的**比例 $p$** 的部分，且該部分加速 $s$ 倍，則整體加速比：

$$
\text{Speedup} = \frac{1}{(1-p) + \dfrac{p}{s}}
$$

關鍵洞見——**被加速比例 $p$ 設下上限**：即使 $s \to \infty$，
$$
\text{Speedup}_{\max} = \frac{1}{1-p}
$$
例：若某段佔執行時間 $p=0.4$，把它變成無限快，整體最多也只快 $1/(1-0.4)\approx 1.67$ 倍。

**工程教訓（"make the common case fast"）**：要優化**佔比大**的部分才有意義；對佔 5% 的部分再怎麼神優化也撼動不了大局。這條定律也是平行運算可擴展性的天花板（序列部分 $1-p$ 限制了多核加速）。

> 量測上常用**幾何平均**彙總多個基準（benchmark）比值，避免被單一項拉偏；SPEC 是常見的標準基準套件。

## 5. 記憶體階層（Memory Hierarchy）與快取（Cache）

### 5.1 為何要階層：速度 vs. 容量的矛盾

理想記憶體要**又大又快又便宜**，但物理上做不到：SRAM 快但貴又小，DRAM 大但慢，磁碟/SSD 超大超便宜但超慢。解法是**疊成金字塔**，靠**局部性原理**讓多數存取打到上層的快記憶體：

```
暫存器 → L1 快取 → L2/L3 快取 → 主記憶體(DRAM) → 磁碟/SSD
（越上越快越小越貴；越下越慢越大越便宜）
```

**局部性（Locality）是一切的基礎**：
- **時間局部性（Temporal）**：剛用過的東西很可能很快再用（迴圈變數）。→ 把它留在快取。
- **空間局部性（Spatial）**：用了某位址，附近位址很可能也要用（陣列、指令流）。→ 一次搬一整塊（block / cache line，如 64 bytes）進來。

階層的精神：**用便宜慢的當後盾，用昂貴快的當前線快取**，整體呈現「接近 L1 的速度 + 接近磁碟的容量」的假象。

### 5.2 快取對應方式（Mapping）

主記憶體一個區塊要放進快取哪一格？三種：
- **直接對應（Direct-mapped）**：每個記憶體區塊只能放在**唯一**一格（用位址中間的 index 位元決定）。查找最快、硬體最省，但兩個都對應到同一格的熱門區塊會**互相踢出（衝突失誤）**。
- **全關聯（Fully associative）**：可放**任何**一格。沒有衝突，但要平行比對所有 tag，硬體昂貴，只用於很小的結構（如 TLB）。
- **N 路組關聯（N-way set associative）**：折衷。把快取分成許多「組（set）」，每組有 N 格；區塊先用 index 對到某一組，組內 N 格任放。**最常用**（如 8-way）。N 越大衝突越少但硬體越貴。

每格存：**有效位元（valid）+ 標籤（tag，位址高位）+ 資料區塊**。位址被切成 **tag | index | block offset** 三段做查找與比對。

### 5.3 讀寫政策與替換

- **命中（hit）/ 失誤（miss）**：要的資料在快取叫命中；不在叫失誤，得往下層抓整塊回來。
- **三種失誤（3C）**：強制性（Compulsory，第一次必失誤）、容量（Capacity，快取太小裝不下工作集）、衝突（Conflict，關聯度不足而互踢）。
- **寫入政策**：
  - **寫穿（Write-through）**：寫快取同時也寫下層。簡單、下層永遠最新；但寫流量大（常配**寫緩衝 write buffer** 緩解）。
  - **寫回（Write-back）**：只寫快取，標記 **dirty**，被替換時才寫回下層。省頻寬，但較複雜、且要處理一致性。
- **寫失誤**：寫配置（write-allocate，先把區塊抓進來再寫，常配寫回）vs. 不配置（no-write-allocate，常配寫穿）。
- **替換策略**：組滿了要踢誰——常用 **LRU（最久未用）**或其近似、隨機。

### 5.4 快取效能與 AMAT

快取失誤會大幅拉高有效 CPI。**平均記憶體存取時間（AMAT）**：
$$
\text{AMAT} = t_{\text{hit}} + \text{miss rate} \times \text{miss penalty}
$$
miss penalty（失誤代價）可能是數十到數百個時脈（要去 DRAM）。這也是為何要多級快取：L2/L3 用來**降低 L1 失誤的代價**。把記憶體效應併入 CPI：
$$
CPI_{\text{實際}} = CPI_{\text{理想}} + \text{每指令記憶體存取數} \times \text{miss rate} \times \text{miss penalty}
$$

直覺：處理器再快，一旦常常 miss 去等 DRAM，整體就被記憶體拖垮——這就是有名的「記憶體牆（memory wall）」。

## 6. 虛擬記憶體（Virtual Memory）

### 6.1 動機與直覺

虛擬記憶體把「快取」概念**再往下推一層**：讓**主記憶體當磁碟的快取**。它解決三件事：
1. **容量假象**：每個程式以為自己擁有一大片連續位址空間（virtual address），即使實體 DRAM 不夠，用磁碟補。
2. **保護與隔離**：每個行程有自己的位址空間，彼此不能亂踩——OS + 硬體強制。
3. **重定位**：程式不必知道自己被放在實體記憶體哪裡，編譯時用虛擬位址即可。

### 6.2 分頁與位址轉換

- 位址空間切成固定大小的**頁（page，如 4KB）**；實體記憶體切成同大小的**頁框（frame）**。
- **頁表（Page Table）**：OS 為每個行程維護的對照表，把**虛擬頁號（VPN）→ 實體頁框號（PFN）**，並含 valid、保護位元、dirty 等。
- 失誤叫**缺頁（page fault）**：要的頁不在 DRAM，得從磁碟搬進來（代價極大，數百萬時脈級，故由 OS 軟體處理，且常觸發行程切換）。
- 因「miss penalty 巨大」，虛擬記憶體的設計選擇都偏向**降低失誤率**：頁夠大（攤平搬移成本）、全關聯式放置、寫回（絕不寫穿磁碟）、用軟體做精細的替換。

### 6.3 TLB：讓轉換不要每次都查頁表

問題：每次存取記憶體都要先查頁表（而頁表本身在記憶體裡）→ 等於每次存取變兩次。解法：**TLB（Translation Lookaside Buffer）**——一個專門**快取最近用過的「VPN→PFN」轉換**的小型、通常高關聯度的快取。

- TLB 命中：一拍內拿到實體位址。TLB 失誤：去走頁表（page table walk，硬體或軟體完成），再填回 TLB。
- 與快取的合作：常見 **VIPT（virtually-indexed, physically-tagged）** 設計，讓 TLB 查找與 L1 快取 index 平行進行，藏住轉換延遲。

**多層概念串起來**：CPU 發出虛擬位址 → TLB 轉成實體位址（TLB miss 就走頁表，頁表 miss 就缺頁去磁碟）→ 拿實體位址查 L1/L2 快取（cache miss 就去 DRAM）。整條路徑就是一連串「快取的快取」，每一層都靠局部性把慢的藏起來。

## 7. I/O 與匯流排（I/O and Buses）

### 7.1 匯流排（Bus）

匯流排是連接 CPU、記憶體、I/O 裝置的**共用線路**，通常分：
- **位址線**（要存取哪裡）、**資料線**（搬什麼）、**控制線**（讀/寫、時序、仲裁）。
- **同步 vs. 非同步**：同步靠共用時脈、快但距離受限；非同步靠交握（handshake）訊號、彈性大。
- **匯流排仲裁（arbitration）**：多個主控（master）想用同一條匯流排時，用 **BR（bus request）/ BG（bus grant）** 等機制決定誰先用，避免衝突。
- 現代多採**點對點高速序列鏈路**（如 PCIe）取代舊式共用平行匯流排，但「位址/資料/控制 + 仲裁」的概念仍在。

### 7.2 CPU 怎麼跟 I/O 溝通：定址方式

- **記憶體映射 I/O（Memory-mapped I/O）**：把裝置的暫存器**映射進一般記憶體位址空間**，用普通 `lw`/`sw` 就能存取裝置。優點：不需特殊指令、定址彈性大（MIPS/ARM/RISC-V 用此法）。代價：佔用位址空間、需注意這些位址不可被快取（或要特別處理）。
- **隔離 I/O / 埠映射（Isolated / Port-mapped I/O）**：I/O 有**獨立位址空間**，用專屬 `in`/`out` 指令存取（x86 傳統做法）。

### 7.3 三種資料傳輸方式（重點對比）

- **輪詢（Polling / Programmed I/O）**：CPU **反覆讀裝置狀態暫存器**等它就緒。簡單，但 CPU 空轉浪費——適合很快或很少的裝置。
- **中斷（Interrupt-driven I/O）**：CPU 下令後**去做別的事**，裝置就緒時發**中斷**打斷 CPU，CPU 跳進中斷服務常式（ISR）處理。不空轉，但每筆資料都要一次中斷 + 進出 ISR 的負擔——適合中速、事件驅動裝置（鍵盤、網路封包）。
- **直接記憶體存取（DMA, Direct Memory Access）**：用專門的 **DMA 控制器**直接在裝置與記憶體間搬**整塊**資料，**CPU 只在開頭設定、結尾收一個中斷**。中間完全不佔 CPU——適合高吞吐裝置（磁碟、網卡、顯示）。代價：DMA 與 CPU 共用記憶體/匯流排，要處理**快取一致性**（DMA 改了 DRAM，CPU 的快取可能還是舊值）與匯流排仲裁。

**直覺排序**：輪詢＝親自站在烤箱前一直看；中斷＝去做別的事，叮一聲再回來；DMA＝請一個助手把整盤端進冰箱，弄完才叫你。負擔依序遞減、適合的資料量依序遞增。

## 8. 與微處理器／數位邏輯的關聯（整科收束）

- **往下接數位邏輯**：資料路徑的每個方塊（暫存器檔、ALU、MUX、管線暫存器、控制 FSM）都是數位邏輯課的組合/序向電路。計算機組織是「把這些零件組成有意義的機器」的那一步。
- **往上接微處理器/系統**：本科建立的處理器 + 記憶體階層 + I/O，就是微處理器（如嵌入式 MCU、應用處理器）的內部組織。再往上是作業系統（管理虛擬記憶體頁表、排程、I/O 驅動）與計算機架構（更激進的 ILP、多核、快取一致性協定）。
- **對通訊背景的提示**：訊號處理（DSP）、基頻處理器、軟體無線電都建立在「能高吞吐跑數值運算的處理器 + 夠快的記憶體頻寬 + DMA 搬資料流」之上——本科的 CPI、管線、快取、DMA 正是評估這類系統效能的語言。

## 9. 碩士延伸（簡短）

- **進階管線與 ILP**：超純量、亂序執行（Tomasulo、重排序緩衝 ROB）、暫存器重命名、推測執行、進階分支預測（TAGE、感知器預測器）。
- **記憶體系統**：多級非阻塞快取、預取（prefetch）、快取一致性協定（MESI/MOESI）、記憶體一致性模型、DRAM 內部時序與排程。
- **平行架構**：多核/眾核、SIMD/向量、GPU 的 SIMT 組織、NUMA、互連網路（network-on-chip）。
- **量化方法**：以 Hennessy & Patterson《Computer Architecture: A Quantitative Approach》為核心，用基準與統計方法做設計取捨。
- **領域專用架構（DSA）**：TPU/NPU、加速器、近記憶體運算。

## 10. 博士研究方向（列表）

- 機器學習加速器與資料流架構（systolic array、稀疏化、量化感知硬體）。
- 近資料／記憶體內運算（Processing-in-Memory, PIM）與打破記憶體牆。
- 新興記憶體技術整合（HBM、CXL 記憶體擴充、非揮發性記憶體階層）。
- 安全微架構：旁路/瞬態執行攻擊（Spectre/Meltdown 類）與防禦、機密運算。
- 亂序核心的能效與可擴展性、推測執行的安全–效能取捨。
- 異質與小晶片（chiplet）整合、封裝級互連與一致性。
- 開放 ISA（RISC-V）上的可重構/可客製化微架構與敏捷硬體設計流程。
- 量子/類比/光學等非傳統計算組織。
