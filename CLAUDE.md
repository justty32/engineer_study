# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

> 注意：本檔案是給 Claude 的工作指引。雖然以繁體中文撰寫，但所有「筆記內容」同樣必須使用繁體中文。

## 這個儲存庫是什麼

這**不是一個軟體專案**，而是一個個人的學習筆記儲存庫。擁有者想對各種工程學科取得廣泛、概論性的認識。這裡的工作是研究與寫作：依據大學課綱用網路搜尋來研究各主題，再寫成概論式的筆記，讓擁有者能熟悉各領域的核心概念。

### 擁有者背景（影響筆記深度）

- 大學就讀**電機工程**，特化於**通訊工程**領域。
- **軟體相關能力很強**；除了軟體之外，其他學科在學時多有被當（基礎並不扎實）。
- 涵義：即使是電機／通訊這類「本科」領域，也不要假設擁有者已有穩固基礎，仍需從基本觀念把脈絡講清楚；軟體相關概念則可以講得精簡。

由於沒有應用程式碼，因此沒有建置（build）、檢查（lint）或測試（test）等指令。最終產出就是筆記本身。

## 硬性要求

- **所有你撰寫的筆記都必須使用繁體中文。** 這是擁有者明確的指示，不可妥協。
- **以文字為主，不放圖。** 許多學科原本仰賴圖表，但擁有者明確表示不要抓取或生成圖片，只寫純文字筆記。目標是先建立初步認識，而非完整參考資料。
- **可以、且鼓勵使用 LaTeX** 來表達公式與數學符號。
- **所有檔案一律使用 UTF-8 編碼**（不要 UTF-16）。覆寫既有檔案時注意工具可能沿用舊編碼，必要時用 `iconv -f UTF-16LE -t UTF-8` 轉換並以 `file <path>` 驗證。

## 筆記慣例

- **每一科都要有中英名詞對照表（中英名詞對照表）**，列出該科核心名詞的繁中與英文對照（許多工程參考資料是英文）。
- 對照表偏好用**獨立檔案**，但若單一科目的對照表**少於 100 行**就視為「小」，應合併到一個**共用對照表檔**（每科一個小節），避免一堆零碎小檔；某科對照表成長到約 100 行以上時再獨立成檔。目前作法：每個領域維護彙整檔；當彙整檔超過約 500 行就切成編號分檔，並保留 `中英名詞對照表.md` 作為「總索引」。電機目前為 `中英名詞對照表.md`（索引）＋ `中英名詞對照表-1～-4.md`（依階段分裝：1 數理基礎、2 電機核心+通訊、3 電力+醫電+晶片、4 控制+天線+嵌入式）。各科筆記開頭的「名詞對照見」連結直接指向其所屬分檔的對應小節。
- **預設風格是概論重點式，但內容多的科目不要省略**。當單檔過大時，依自然子主題切成多個檔案，命名用 `上/下`、`上/中/下`、或 `1/2/3/4…`；各部之間互相連結。
- **共用對照表自身過長**（約 500 行以上）時，把它切成 `中英名詞對照表-1.md`、`-2.md`……，各檔內為若干完整科目。

## 專案組織與擴充規範

進一步的擴充規則（共用基礎抽出、知識點延伸、段落擴充、extra 整合、專題式學習）見 [roadmap.md](roadmap.md)。AI 在這些時機可主動**提出建議**，實際大規模搬移得到擁有者同意後再執行。

## 內容模型

筆記依學術深度分層，請依當前涵蓋的層級採用對應的深度：

- **大學（University）層級** — 主要重點。確實涵蓋標準課綱，這是擁有者真正想學的部分。
- **碩士（Master's）層級** — 各子領域只做簡短介紹。只有當擁有者對特定主題表達興趣時，才針對該主題做深入的網路研究。
- **博士（PhD）層級** — 以主題／研究方向的「列表」呈現，不寫成長篇說明。

## 工作流程

- 從某領域典型的大學課綱出發來建立學習計畫，再針對其中的知識點研究並撰寫筆記。
- 擁有者可能會先請你規劃學習計畫，再依計畫填入筆記。
- **起始領域，依序為：** 電機（Electrical Engineering）、電子（Electronics）、機械（Mechanical Engineering）。
