# 工程圖學與電腦輔助設計（Engineering Drawing & CAD）

> 名詞對照見 ../中英名詞對照表.md

工程圖學是機械工程師之間的「共同語言」。一個零件從設計者腦中的構想，到加工廠的工人手中、到品管的量測台上，中間靠的不是口頭描述，而是一張規範嚴謹、人人讀法一致的工程圖。它的核心任務是：用平面（2D）上的線條與符號，**毫無歧義地**把一個三維（3D）實體的「形狀、尺寸、精度、表面狀態」完整傳達出去。對軟體背景的人來說，可以把工程圖想成一份「規格書 / 介面契約」——它定義了零件必須滿足的所有可量測條件，至於用什麼機台、什麼程式去做出來，是製造端的自由。本篇重點放在這套規範與觀念，而不是某套 CAD 軟體的按鈕操作。

## 0. 本科在本系的定位

- **它是什麼**：把 3D 幾何＋公差＋表面要求，編碼成一張標準化、可被任何人一致解讀的 2D 工程圖（或 3D 模型加註記）。它同時包含「投影幾何」（怎麼把立體攤平成視圖）與「標註規範」（怎麼寫尺寸、公差、符號）。
- **往上游銜接**：需要立體空間想像、基本幾何，以及一點點向量 / 座標觀念（CAD 內部就是座標幾何）。
- **往下游銜接**：是後續所有設計與製造科目的輸出格式。[機械設計](../04-機構與機械設計/10-機械設計.md)算出來的尺寸與安全係數，最後要落成有公差的工程圖；[製造與製程](../04-機構與機械設計/11-製造與製程.md)則依圖上的尺寸、公差、表面粗糙度來決定用什麼加工方法、能不能做得到、成本多少。公差配合更直接影響[機動學](../04-機構與機械設計/09-機動學.md)裡軸承與配合件能否正常運轉。
- **核心心法**：圖面上**每一條線、每一個數字、每一個符號都有唯一解讀**，沒有「看圖的人自己猜」的空間。歧義就是不良圖。設計的本意（design intent）必須被「可量測的條件」完整表達。

## 1. 投影原理：怎麼把立體攤平

### 1.1 為什麼要正投影

人眼看到的立體有透視（遠小近大），但工程圖不能用透視——因為透視會讓尺寸失真，量不準。工程圖用的是**正投影（orthographic projection）**：想像投影線彼此平行、且垂直於投影面，把物體「壓扁」到平面上。這樣同一深度的特徵在圖上保持等比例，可以直接量測。一個物體通常要好幾個方向的正投影視圖（前、上、側）合起來，才能完整描述。

### 1.2 第一角法與第三角法

把物體想成放在一個玻璃箱裡，往六個面投影，再把箱子攤開成平面。差別在於「物體相對於觀察者與投影面的順序」：

- **第三角法（third angle，ISO 中的代號等同美規 ANSI 慣用）**：投影面在物體與觀察者之間，像隔著玻璃描圖。攤開後，**右視圖放在前視圖的右邊、上視圖放在前視圖的上方**——視圖位置與你實際繞著看的直覺一致。
- **第一角法（first angle，歐洲、ISO 傳統與台灣 CNS 常用）**：物體在觀察者與投影面之間，投影「穿過物體投到後方的面」。攤開後，**右視圖跑到前視圖的左邊、上視圖跑到下方**——位置與直覺相反。

兩者描述的是同一個物體，只是視圖擺放位置左右上下對調。因此每張圖都必須在標題欄附近標示採用哪一角法（通常用一個截頭圓錐的標準符號表示），否則讀圖者可能把左右弄反，造出鏡像零件。

| 項目 | 第一角法（First Angle） | 第三角法（Third Angle） |
| --- | --- | --- |
| 想像模型 | 物體在投影面前，光穿過物體 | 物體隔著玻璃，向觀察者側投影 |
| 右視圖位置 | 前視圖左側 | 前視圖右側 |
| 上視圖位置 | 前視圖下方 | 前視圖上方 |
| 主要使用區 | 歐洲、ISO、台灣 CNS | 美國 ANSI、日本部分產業 |

## 2. 視圖與其配置

### 2.1 三視圖與視圖選擇

最常用的是**前視圖（front view，主視圖）＋上視圖（top view）＋右側視圖（right side view）**三視圖。前視圖要選最能表達物體特徵、且擺放最自然（加工或使用姿態）的方向。原則是「用最少的視圖把物體講清楚」：簡單回轉體可能一個視圖加直徑符號 $\varnothing$ 就夠；複雜件才需要更多視圖。視圖之間靠投影對齊——上視圖的寬度、前視圖的寬度、側視圖的高度等必須對齊，這就是「視圖配置」的紀律。

### 2.2 輔助視圖、局部視圖、詳圖

- **輔助視圖（auxiliary view）**：當斜面不平行於任何主投影面時，正投影會失真。此時沿垂直於斜面的方向再投一個視圖，才能看到斜面真實形狀與尺寸。
- **局部視圖（partial view）**：只畫需要的局部，避免重複與雜亂。
- **詳圖 / 放大圖（detail view）**：把太小看不清的局部圈起來放大（例如以比例 5:1）標示。

## 3. 剖視圖：看見內部

當零件內部有孔、槽、肉厚變化時，外形視圖只能用虛線表示看不見的邊，虛線一多就亂。**剖視圖（section view）**的作法是假想用一個切割面把物體切開、移去前半，畫出切到的斷面與後方可見輪廓。被實體切到的斷面要打**剖面線（hatching，等距 45° 斜線）**，不同零件用不同方向或間距區分。

常見類型：

- **全剖（full section）**：整個切過去。
- **半剖（half section）**：對稱件只切一半，一半看外形、一半看內部，省一個視圖。
- **局部剖 / 破斷剖（broken-out section）**：只剖局部，用不規則破斷線界定範圍。
- **旋轉剖 / 移出剖（revolved / removed section）**：表達肋、輪輻、軸的斷面形狀。

慣例：像肋（rib）、輻條、螺栓、軸這類實心或標準件，即使被縱向切到，習慣上**不打剖面線**，以免誤解為實心塊。

## 4. 尺寸標註：寫上「多大、在哪」

視圖給形狀，尺寸給大小與位置。良好標註的原則：

- **不重複、不遺漏**：每個尺寸只標一次（避免互相矛盾），但所有製造必需的尺寸都要有。
- **功能優先**：依設計與裝配的功能基準來標，而不是隨手從某條邊量。基準（datum）選定後，相關尺寸盡量從同一基準量出，減少累積誤差（鏈式標註會疊加公差）。
- **符號**：直徑 $\varnothing$、半徑 $R$、球面 $S\varnothing$、沉頭孔、柱坑、深度等都有專用符號；螺紋以代號標（如 $\mathrm{M10\times1.5}$）。
- **不直接量測製造程序**：圖只描述「成品要長怎樣」，不規定加工順序。

## 5. 公差與配合：精度的語言

### 5.1 為什麼要公差

世界上沒有「剛好 50.000 mm」的加工。一旦容許誤差、又要保證能裝得起來、能運轉，就必須規定每個尺寸的**容許範圍**。公差（tolerance）＝上下限之差。標法可為：上下偏差（$50^{+0.05}_{-0.02}$）、對稱（$50\pm0.03$）、或用 ISO 配合代號。公差越緊，加工越貴，所以「該緊的緊、不重要的鬆」是成本控制的關鍵。

### 5.2 ISO 286 孔軸制與配合

ISO 286 用「**基本尺寸 + 一個字母 + 一個數字**」描述一個尺寸的公差帶，例如 $\varnothing50\,\mathrm{H7}$、$\varnothing50\,\mathrm{g6}$：

- **字母＝基本偏差（fundamental deviation）**：決定公差帶相對於基本尺寸的「位置」（往上偏還是往下偏）。孔用大寫、軸用小寫。
- **數字＝公差等級 IT（IT grade）**：決定公差帶的「寬度」。數字越小越精密（IT01 最精密，IT16 最粗）。一般配合常用 IT6～IT8。

配合（fit）＝同一基本尺寸下，孔與軸兩個公差帶的相對關係，分三種：

| 配合種類 | 關係 | 結果 | 例 |
| --- | --- | --- | --- |
| 餘隙配合（clearance fit） | 孔恆大於軸 | 永遠有間隙，可自由轉動 / 滑動 | H7/g6、H7/h6 |
| 過渡配合（transition fit） | 視實際尺寸而定 | 可能微隙也可能微緊，定位準 | H7/k6、H7/n6 |
| 過盈配合（interference fit） | 軸恆大於孔 | 必須壓入，靠摩擦固定 | H7/p6、H7/s6 |

軸字母規律可記為：$a\sim h$ 給餘隙、$j\sim n$ 給過渡、$p\sim z$ 給過盈。

兩種基準制：

- **基孔制（hole basis）**：固定孔為 $\mathrm{H}$（下偏差 $=0$），靠改變軸字母得到不同配合。最常用，因為孔多用標準鉸刀 / 量規，固定孔較省工具。
- **基軸制（shaft basis）**：固定軸為 $\mathrm{h}$（上偏差 $=0$），改變孔字母。適合用標準現成軸料（如冷拉圓棒）的場合。

## 6. 幾何尺寸與公差（GD&T）概念

只標長寬高的「尺寸公差」管不到形狀與相對位置：一根軸即使直徑合格，也可能彎曲、橢圓或不同心。**GD&T（幾何尺寸與公差，依 ASME Y14.5 或 ISO GPS）**用一套符號專門控制幾何特性，並引入**基準（datum）**作為量測參考系。

核心觀念：

- **基準（datum）**：以零件上實際的面 / 軸 / 中心為「量測時的座標原點與參考方向」，建立基準參考框架，確保設計、製造、檢驗對齊一致。
- **特徵控制框（feature control frame）**：一格一格的方框，依序寫「幾何特性符號 ｜ 公差值（前面常加 $\varnothing$ 表圓形公差帶）｜ 基準字母」，例如「位置度 $\varnothing0.1$ A B C」。

ASME Y14.5 共 14 個幾何特性符號，分五類：

| 類別 | 符號（特性） | 是否需要基準 |
| --- | --- | --- |
| 形狀（form） | 真直度、真平度、真圓度、圓柱度 | 不需基準 |
| 輪廓（profile） | 線輪廓度、面輪廓度 | 視情況 |
| 方位（orientation） | 平行度、垂直度、傾斜度 | 需基準 |
| 位置（location） | 位置度、同心度、對稱度 | 需基準 |
| 偏轉（runout） | 圓偏轉、總偏轉 | 需基準 |

GD&T 的價值：相較於傳統 $\pm$ 標註，它能更精準表達「設計真正在意什麼」，常可放寬非關鍵公差（降低成本）、又把關鍵裝配特性鎖得更明確，並支援**最大實體條件（MMC）**等修飾符，用以換取額外公差（bonus tolerance）。

## 7. 表面粗糙度與表面符號

零件表面再怎麼磨也有微觀凹凸，影響密封、摩擦、疲勞與外觀。表面粗糙度用參數量化，最常用 **$Ra$（算術平均粗糙度）**，單位 µm；數值越小越光滑。圖上用表面符號（一個帶分支的勾形記號）標示，並可註明粗糙度上限、加工方式（如車、磨）、紋路方向等。觀念上：該密封 / 高速旋轉的配合面要求低 $Ra$（光滑、昂貴），非接觸的粗胚面則放寬，與公差一樣是「成本 vs 功能」的權衡。

## 8. 製圖規範（ISO / CNS / ANSI）

工程圖之所以能跨公司、跨國通用，靠的是標準：

- **ISO**：國際標準（含 ISO 128 製圖、ISO 286 公差配合、ISO 1101 幾何公差、GPS 體系等），歐洲與多數地區採用，慣用第一角法。
- **CNS**：台灣國家標準，多承襲 ISO，亦以第一角法為主。
- **ANSI / ASME**：美國標準（含 ASME Y14.5 的 GD&T），慣用第三角法。

規範內容涵蓋：圖紙尺寸（A0～A4）、線型（粗實線=可見輪廓、細實線=尺寸線、虛線=隱藏線、鏈線=中心線等各有意義）、字體、比例、標題欄與零件表（BOM）、公差預設值等。線型的「意義」是讀圖的基本功——一條線是什麼線，直接決定它代表的幾何含義。

## 9. CAD：從 2D 圖板到 3D 參數模型

CAD（電腦輔助設計）讓上述工作從手繪走向電腦。對軟體背景的人，這部分觀念極快上手，操作步驟可略，重點在「建模心法」：

### 9.1 特徵造型與參數化

現代 3D CAD（SolidWorks、Creo、Fusion、CATIA 等）採**特徵式造型（feature-based modeling）**：先畫 2D 草圖（sketch），再用伸長（extrude）、迴轉（revolve）、掃掠（sweep）、放樣（loft）等動作長出立體，後續用圓角、倒角、孔、陣列等特徵堆疊。模型是一棵**特徵樹**，按時間順序記錄每一步。

**參數化（parametric）**是關鍵：尺寸是可命名的變數，特徵間有父子依賴，改一個尺寸，整個模型自動重算更新。這正是軟體人熟悉的「宣告式 / 相依重算」思維，類似試算表或響應式資料流。

### 9.2 約束、組合件、出圖

- **約束（constraints）**：草圖用幾何約束（相切、平行、共線、對稱）＋尺寸約束鎖定，目標是讓草圖「完全定義」（fully defined）——沒有自由度殘留，形狀唯一確定。約束不足的草圖會「飄」。
- **組合件（assembly）**：多個零件用配合關係（mate：貼合、同軸、距離、角度）組起來，模擬實際裝配，可檢查干涉與運動。
- **工程圖出圖（drawing）**：3D 模型可自動投影出標準視圖、剖視圖、爆炸圖並半自動帶出尺寸，再加上公差、GD&T、表面符號與標題欄。也就是說，前面 1～8 節的規範，最終仍是 CAD 出圖階段要套用的——**3D 建得再漂亮，沒有正確標註的工程圖仍無法製造**。

## 與其他科目的關聯

- [機械設計](../04-機構與機械設計/10-機械設計.md)：設計計算出的尺寸與安全係數，最終以有公差的工程圖表達；公差配合直接源於本科。
- [製造與製程](../04-機構與機械設計/11-製造與製程.md)：加工方法、可行性與成本，取決於圖上的公差等級與表面粗糙度。
- [機動學](../04-機構與機械設計/09-機動學.md)：軸承、銷、軸孔等運動配合，依賴正確的配合種類（餘隙 / 過渡 / 過盈）選擇。
- [材料力學](../02-固體力學/04-材料力學.md)：圖上的應力集中處（孔、圓角）標註與表面狀態，關係到疲勞壽命。

## 碩士層級延伸（簡短）

- **參數化與知識驅動設計**：以規則 / 公式驅動模型族（design table、配置），自動產生變體。
- **PLM / PDM（產品生命週期 / 資料管理）**：版本控制、變更管理、協同設計，概念上類似程式碼的版控與 CI。
- **逆向工程（reverse engineering）**：以 3D 掃描點雲重建 CAD 模型。
- **生成式設計與拓樸最佳化（generative design / topology optimization）**：給定載重與限制，讓演算法生成最省材料的形狀。
- **MBD（model-based definition）**：把公差、GD&T、註記直接掛在 3D 模型上，逐步取代 2D 工程圖。
- **GPS 與公差堆疊分析（tolerance stack-up）**：以統計 / 最壞情況分析裝配公差累積。
